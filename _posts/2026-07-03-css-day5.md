웹개발 수업 5일차. 반응형(@media), CSS 변수, 효과 4종, 가상 요소, CSS 리셋을 배우고 미니 프로젝트로 CSS 과정을 완주했다. HTML+CSS 전 기법이 한 페이지에 다 들어간 프로필 카드가 첫 결과물. 심화에서는 수업에 나온 개념 세 가지(@media, :root, box-shadow)를 MDN 공식 문서로 직접 파봤다.

## 1. 수업 내용 요약

### 1-1. 화면 크기별 스타일 — media query

화면 폭에 따라 다른 CSS를 적용하는 반응형의 핵심. 한 CSS 파일로 PC·모바일을 동시 대응한다.

```css
/* 기본(넓은 화면): 카드를 가로로 나란히 */
.box { display: flex; flex-direction: row; }

/* 화면 폭이 600px 이하일 때만 아래 규칙 적용 (모바일) */
@media (max-width: 600px) {
  .box { flex-direction: column; }   /* 세로로 쌓기 */
}
```

`@media (...)` 안의 규칙은 조건을 만족할 때만 기존 규칙을 **덮어쓴다**. 브라우저 창을 줄였다 늘이며 가로↔세로 전환을 눈으로 확인했다.

### 1-2. CSS 변수 (custom properties · :root · var())

값에 이름표를 붙여 한 곳에서 정의하고 여러 곳에서 꺼내 쓰는 기능.

```css
/* 1) 정의 — :root 안에 --이름: 값 */
:root {
  --main-color: #ec4899;
  --gap: 12px;
}

/* 2) 사용 — var(--이름) */
.btn { background: var(--main-color); }
.row { gap: var(--gap); }
```

- 변수 이름은 항상 `--`(대시 두 개)로 시작
- 버튼 2개가 같은 `--main-color`를 쓰면 → **`:root` 한 줄만 고쳐도 둘 다 동시에 바뀜**. 유지보수의 핵심.
- 지난 시간 배운 `rem`(html 한 곳 기준)과 같은 철학 — 반복되는 값은 한 곳에 모아 관리한다.

### 1-3. 그림자·둥근모서리·투명도·전환 — 효과 4종 세트

카드 하나에 효과 4종을 모두 적용해봤다.

```css
.card {
  border-radius: 16px;                          /* ① 모서리 둥글게 */
  opacity: 0.85;                                /* ② 살짝 비치게 (1=불투명, 0=투명) */
  box-shadow: 0 6px 18px rgba(0,0,0,0.25);      /* ③ 그림자 (x·y·번짐·색) */
  transition: all 0.3s ease;                    /* ④ 바뀔 때 0.3초 스르륵 */
}
.card:hover {
  opacity: 1;                                   /* 또렷하게 */
  transform: translateY(-8px);                  /* 위로 8px 떠오름 */
  box-shadow: 0 16px 32px rgba(255,105,180,0.5); /* 그림자 진하게 */
}
```

`transition` 덕에 hover 시 카드가 부드럽게 떠오른다. 숫자(`0.3s`, `-8px`)만 바꿔 속도·높이 조절 가능.

### 1-4. 가짜 요소로 장식 추가 — ::before · ::after

HTML을 건드리지 않고 콘텐츠 앞(`::before`)과 뒤(`::after`)에 장식을 끼워 넣는 **가상 요소**. 실제 DOM에는 없는 가짜 요소라서 **`content` 속성이 반드시 있어야** 화면에 나타난다.

```css
.star-title::before { content: "★ "; color: gold; }   /* 제목 앞에 별 */
.quote::before { content: "“ "; }                      /* 인용문 앞 따옴표 */
.quote::after  { content: " ”"; }                      /* 인용문 뒤 따옴표 */
```

**왜 쓸까? — 장식용 빈 `<span>`을 HTML에 직접 넣지 않고 CSS 한 곳에서 처리:**

1. HTML이 깔끔하게 유지됨 (장식 태그가 늘어나지 않음)
2. 한 줄만 수정하면 같은 클래스 전체에 반영 (유지보수 ↑)
3. 별·따옴표는 "내용이 아님" → 스크린리더가 읽지 않음 (접근성 ↑)

실전 예시가 와닿았다 — 폼의 필수 입력 `*` 표시. 필드 20개에 `<span class="required">*</span>`를 20번 쓰는 대신:

```css
.required::after { content: " *"; color: red; }
```

HTML엔 `class="required"`만 붙이면 끝. 별 모양·색 변경도 CSS 한 줄, 필수↔선택 전환도 class 추가·제거만으로 끝난다.

⚠️ 단, 진짜 내용(검색·SEO·복사가 필요한 글)은 `content`로 넣지 말 것 — 순수 장식만.

### 1-5. CSS 리셋 — 브라우저 기본 여백 없애기

브라우저는 `body`·`h1`·`p` 등에 **기본 여백을 미리 넣어둔다**. 그래서 화면 가장자리에 흰 틈이 생기고, 브라우저마다 값도 조금씩 다르다. 작업 시작 전 이 기본값을 0으로 초기화하는 게 **CSS 리셋** — 실무 거의 필수.

```css
/* main.css 맨 위에 먼저 */
* {                          /* * = 모든 요소 (와일드카드) */
  margin: 0;
  padding: 0;
  box-sizing: border-box;    /* 크기 계산 방식 변경 */
}
```

**`box-sizing: border-box`가 핵심.** `width: 100px; padding: 20px`을 줬을 때:

| 방식 | 실제 폭 | 특징 |
|------|---------|------|
| `content-box` (기본) | 100 + 20 + 20 = **140px** | padding이 더해져 삐져나감 |
| `border-box` | **100px** 그대로 | 내가 정한 width가 최종 크기 |

border-box면 padding·border를 줘도 박스가 안 커져서 레이아웃이 안 틀어진다. 그래서 리셋에 거의 항상 포함.

### 1-6. 미니 프로젝트 — HTML + CSS 전 기법 종합 프로필 카드 🎉

CSS 01~18의 기법을 한 페이지에 모은 최종본. 들어간 기법들:

- `:root` CSS 변수로 테마 색·간격 한 곳 관리
- `position: sticky` 헤더 (스크롤해도 상단 고정) + `relative` 부모 & `absolute` NEW 뱃지
- `display: grid` 카드 갤러리 + `@media`로 좁은 화면에서 1칸 전환 (반응형)
- `box-shadow` · `border-radius` · `transform` · `transition` hover 효과
- `::before` 별 장식 + `linear-gradient` 배경
- 맨 위엔 CSS 리셋

**가장 인상 깊었던 것:** `--main` 변수 하나만 바꾸면 헤더·카드·별 장식까지 테마 전체가 한 번에 바뀐다. 변수로 설계하는 이유를 몸으로 이해했다.

## 2. 심화 학습 — MDN 공식 문서로 파고들기

수업에 나온 세 가지를 MDN에서 직접 찾아 요약했다. 수업 밖 내용도 꽤 있었다.

### 2-1. @media — 화면 크기만이 아니었다

`@media`는 CSS의 **@규칙(at-rule)** 중 하나로, 특정 미디어 쿼리 조건을 만족하는 장치에서만 스타일을 적용한다.

```css
@media screen and (min-width: 900px) {
  article { padding: 1rem 3rem; }
}
```

수업에선 화면 폭(`max-width`)만 배웠는데, 용도가 더 넓다:

- **반응형 웹 디자인** — 화면 크기별 스타일 (수업에서 배운 것)
- **인쇄용 스타일** — `@media print` (인쇄할 때만 적용, 예: 메뉴 숨기기)
- **사용자 설정 감지** — 다크모드, 애니메이션 감소(멀미 방지) 등

**최신 문법(Media Queries Level 4)**에서는 비교 연산자로 더 간결하게 쓸 수 있다:

```css
@media (height > 600px) {
  body { line-height: 1.4; }
}
```

브라우저 호환성은 널리 지원되는 안정적인 기능.

### 2-2. :root — html 선택자와 뭐가 다르지?

`:root`는 문서 트리의 **루트 요소**를 선택하는 의사 클래스(pseudo-class)다. HTML 문서에서 루트는 `<html>`이므로 사실상 `html` 선택자와 동일하게 동작하는데, 차이가 하나 있다 — **`:root`가 명시도(specificity)가 더 높다.**

```css
:root {
  --main-color: hotpink;
  --pane-padding: 5px 42px;
}
```

가장 많이 쓰이는 용도는 역시 **전역 CSS 변수 선언**. `:root`에 선언하면 문서 전체 어디서든 `var(--main-color)`로 꺼내 쓸 수 있다. 2015년부터 널리 지원되는 안정적인 기능.

> 요소 선택자(html)보다 클래스류가 우선한다는 명시도 규칙이 여기에도 적용된다는 게 재밌었다. 4일차에 배운 "구체적인 선택자가 이긴다"의 연장선.

### 2-3. box-shadow — 값이 5개나 있었다

수업에선 `0 6px 18px 색` 형태(x·y·번짐·색)만 썼는데, 전체 문법은 이렇다:

```css
box-shadow: <x-offset> <y-offset> <blur-radius> <spread-radius> <color>;
```

| 값 | 필수? | 역할 |
|-----|------|------|
| offset-x, offset-y | 필수 | 그림자의 가로/세로 위치 |
| blur-radius | 선택 | 흐림 정도 — 클수록 크고 흐릿 |
| **spread-radius** | 선택 | 그림자 크기 확장/축소 (수업에 안 나온 4번째 값!) |
| color | 선택 | 생략 시 부모의 `color` 값 사용 |
| **inset** | 선택 | 그림자를 요소 **안쪽**으로 (기본은 바깥) |

**쉼표로 구분하면 그림자를 여러 개 겹칠 수도 있다:**

```css
box-shadow: 10px 5px 5px red;                 /* 기본 그림자 */
box-shadow: inset 5em 1em gold;               /* 안쪽 그림자 */
box-shadow: 3px 3px red, -1em 0 0.4em olive;  /* 그림자 2개 겹치기 */
```

알아두면 좋은 참고사항:
- `border-radius`가 설정된 요소는 그림자도 같이 둥글게 나온다
- 여러 그림자를 쓰면 **먼저 쓴 것이 위에** 표시된다
- 4일차 네온 카드에서 흐림 반경을 크게 줬던 게 blur-radius였고, spread까지 쓰면 빛 번짐을 더 정교하게 조절할 수 있겠다
