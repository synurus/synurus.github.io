웹개발 수업 2일차. 어제 배운 기본 태그에 이어서 표, 폼, 영상/지도 넣기, 그리고 페이지 구조를 잡는 class·id·div·span과 시맨틱 태그까지. HTML의 나머지 대부분을 훑은 날이다.

## 1. 수업 내용 요약

### 1-1. 번호 목록, 줄바꿈, 가로선 (ol, br, hr)

어제 배운 `<ul>`은 점(•) 목록. 순서가 중요할 때(요리 순서, 등수)는 **`<ol>`**을 쓰면 1·2·3 번호가 자동으로 붙는다. 안에 들어가는 `<li>`는 똑같다.

```html
<ol>
  <li>일어나기</li>   <!-- 자동으로 1. -->
  <li>세수하기</li>   <!-- 자동으로 2. -->
</ol>

<p>첫째 줄<br>둘째 줄</p>  <!-- br = 문단 안에서 줄바꿈 -->
<hr>                        <!-- hr = 내용을 나누는 가로선 -->
```

`<br>`과 `<hr>`은 어제 배운 `<img>`처럼 닫는 태그가 없는 **빈 요소**다.

### 1-2. 표 만들기 + 칸 합치기 (table, tr, th, td, colspan, rowspan)

시간표·성적표처럼 칸이 나뉜 표. 가로 줄을 위에서 아래로 차곡차곡 쌓는 방식이다.

| 태그 | 역할 |
|------|------|
| `<table>` | 표 전체를 담는 상자 |
| `<tr>` | 가로 한 줄 (row) |
| `<th>` | 제목 칸 (굵게·가운데 정렬) |
| `<td>` | 보통 내용 칸 |

```html
<table border="1">                        <!-- border="1" 없으면 칸 선이 안 보임 -->
  <thead>                                 <!-- 표 머리(제목 줄) -->
    <tr><th colspan="3">성적표</th></tr>  <!-- colspan=3 → 가로 3칸 합침 -->
  </thead>
  <tbody>                                 <!-- 표 몸통(내용 줄) -->
    <tr><td rowspan="2">언어</td><td>국어</td><td>90</td></tr>  <!-- rowspan=2 → 세로 2줄 합침 -->
    <tr><td>영어</td><td>85</td></tr>     <!-- 위 칸이 내려와서 2칸만 적음 -->
  </tbody>
</table>
```

- **`colspan`** = 가로로 N칸 합치기 / **`rowspan`** = 세로로 N칸 합치기
- `rowspan`을 쓴 다음 줄은 그 칸이 이미 채워져 있어서 칸 수를 하나 줄여 적는다. (처음에 여기서 헷갈렸다!)

### 1-3. 폼: 입력 받기 (form, label, input, button)

회원가입·검색창처럼 사용자에게 입력을 받는 태그들. 입력칸들을 `<form>` 상자로 묶는다.

```html
<form>
  <label>이름: <input type="text" placeholder="이름 입력"></label>
  <label>나이: <input type="number"></label>
  <button type="submit">보내기</button>
</form>
```

- `<label>` = 입력칸 옆 설명 글자
- `<input>` = 입력 칸 (빈 요소)
- `placeholder` = 칸이 비었을 때 보이는 흐린 안내 글자

**input의 type만 바꾸면 칸 모양이 달라진다:**

| type / 태그 | 용도 |
|------|------|
| `type="email"` | 메일 칸 (@ 자동 검사) |
| `type="password"` | 점(••)으로 가려지는 칸 |
| `type="checkbox"` | 여러 개 고르기 (네모) |
| `type="radio"` | 하나만 고르기 (동그라미), `name`이 같으면 한 묶음 |
| `type="date"` | 날짜 고르기 (달력) |
| `<textarea>` | 여러 줄 긴 글 칸 |
| `<select><option>` | 드롭다운 (펼쳐서 하나 고르기) |

### 1-4. iframe: 영상과 지도 박기

`<iframe>` = 내 페이지 안에 **다른 페이지를 작은 네모 창으로 끼워 넣기**. 유튜브 영상과 구글 지도를 그대로 박을 때 쓴다.

```html
<!-- 유튜브: 일반 주소 X, embed 주소를 써야 뜬다 -->
<iframe width="320" height="180"
  src="https://www.youtube.com/embed/영상ID"></iframe>

<!-- 구글 지도: q=장소, output=embed -->
<iframe width="320" height="220"
  src="https://www.google.com/maps?q=서울시청&amp;output=embed"></iframe>
```

- 유튜브 → **공유 → 퍼가기(embed)** 를 누르면 iframe 코드를 그대로 준다.
- 구글 지도 → **공유 → 지도 퍼가기** 로 받은 코드가 정확하다. `pb=…` 부분은 위치·확대값이 담긴 자동 생성 주소라 손대지 말고 통째로 붙여넣기.
- 주소 속 `&`는 HTML 특수문자라서 정석은 `&amp;`로 적는다. (어제 배운 엔티티가 여기서 또 나왔다.)

### 1-5. class와 id: 태그에 이름표 달기

나중에 CSS·JS가 요소를 콕 집어 꾸미거나 움직일 수 있도록 이름표를 다는 것. **CSS로 가는 다리**다.

| 구분 | class | id |
|------|-------|-----|
| 비유 | 반 이름표 | 학번 |
| 중복 | 같은 값 여러 번 OK | 페이지에 딱 하나만 |
| 주 용도 | 여러 요소를 묶어 한 번에 꾸미기 | 메뉴 점프(`#id`), label 연결 |
| CSS에서 부를 때 | `.이름` (점) | `#이름` (샵) |

```html
<p class="note">첫째 줄</p>
<p class="note">둘째 줄</p>   <!-- 나중에 CSS로 .note 하나만 꾸미면 둘 다 바뀜 -->

<h1 id="top">맨 위 제목</h1>  <!-- 같은 id 값 두 번 쓰면 안 됨 -->
```

### 1-6. div와 span: 뜻 없는 만능 상자

둘 다 의미 없는 빈 상자로, 여러 요소를 묶어서 한꺼번에 꾸밀 때 쓴다. 차이는 자리 차지 방식.

- **`<div>`** = 블록 상자. 한 줄을 통째로 차지 (위→아래로 쌓임)
- **`<span>`** = 인라인 상자. 줄을 안 바꾸고 글자 사이에 끼어듦

```html
<div class="card">          <!-- 사진·이름·가격을 상품 카드 한 덩어리로 묶기 -->
  <img src="shoes.png" alt="신발">
  <h3>운동화</h3>
  <p>59,000원</p>
</div>

<p>글자 중 <span>여기만</span> 강조</p>  <!-- 문장 속 일부만 묶기 -->
```

**고르는 순서:** ① 뜻에 맞는 시맨틱 태그가 있으면 그걸 먼저 → ② 마땅한 게 없으면 한 줄 통째는 `div`, 글자 일부는 `span`.

### 1-7. 파일 경로: 이미지가 안 뜨는 이유 1위

경로 = 그 파일이 어느 폴더에 있는지 알려주는 길 안내.

```html
<img src="./img/cat.png">           <!-- ./ = 지금 폴더 안의 img 폴더 -->
<img src="../logo.png">             <!-- ../ = 한 칸 위 폴더 -->
<img src="https://picsum.photos/80"> <!-- https:// = 인터넷 주소 (절대경로) -->
```

이미지가 안 뜨면 십중팔구 경로 문제. `./`와 `../`의 차이를 확실히 기억하자.

### 1-8. 링크 더 쓰기 (a 속성)

`<a>`에 붙이는 속성마다 하는 일이 다르다.

```html
<a href="https://google.com" target="_blank">새 탭에서 열기</a>
<a href="#top">같은 페이지의 id="top" 위치로 점프</a>
<a href="mailto:hi@me.com">메일 쓰기 창 열기</a>
<a href="tel:01012345678">전화 걸기 (폰에서)</a>
```

`href="#top"`은 어제 배운 `id`와 짝이다. 페이지가 길 때 "맨 위로" 버튼을 이걸로 만든다.

### 1-9. 글자 꾸미기 태그

| 태그 | 효과 |
|------|------|
| `<em>` | 기울임 강조 |
| `<mark>` | 형광펜 |
| `<small>` | 작게 |
| `<code>` | 코드 글씨 |
| `<blockquote>` | 인용 덩어리 |
| `<sub>` / `<sup>` | 아래첨자(H<sub>2</sub>O) / 위첨자(x<sup>2</sup>) |

### 1-10. 접는 상자 (details, summary)

JavaScript 없이 클릭만으로 펴고 접는 상자. FAQ나 "더보기"에 쓴다.

```html
<details>
  <summary>정답 보기</summary>   <!-- 항상 보이는 제목(클릭 버튼) -->
  <p>여기는 눌러야 보여</p>      <!-- 펼쳐야 보이는 내용 -->
</details>
```

### 1-11. 의미를 담은 구역 태그 (nav, section, article, aside)

어제 배운 header·main·footer에 이어 시맨틱 구역 태그 4개 추가. 자세한 정리는 아래 심화 학습에서.

```html
<header>
  <nav>홈 · 소개 · 연락처</nav>
</header>
<main>
  <section>
    <article>첫 번째 글 내용</article>
  </section>
</main>
<aside>곁다리 정보(사이드바)</aside>
<footer>ⓒ 2026 내 사이트</footer>
```

## 2. 어려웠던 것: 심화 학습

### 시맨틱 태그 7종 총정리

이틀에 걸쳐 시맨틱 태그가 7개나 나와서 한 번에 정리했다.

| 태그 | 역할 |
|------|------|
| `<header>` | 페이지나 섹션의 머리말 (로고, 제목, 메뉴 등) |
| `<nav>` | 내비게이션(메뉴) 링크 모음 |
| `<main>` | 문서의 핵심 콘텐츠, **페이지당 하나만** |
| `<section>` | 주제별로 묶인 콘텐츠 영역 |
| `<article>` | 독립적으로 의미를 갖는 콘텐츠 (블로그 글, 뉴스 기사 등) |
| `<aside>` | 본문과 간접적으로 관련된 부가 콘텐츠 (사이드바, 광고 등) |
| `<footer>` | 페이지나 섹션의 꼬리말 (저작권, 연락처 등) |

### 순서대로 시맨틱 구조가 이루어진다

이 태그들은 낱개로 외우는 게 아니라, **페이지 전체 구조**로 이해해야 한다.

```
<body>
├── <header>          ← 맨 위 머리말
│    └── <nav>        ← 메뉴는 보통 header 안에
├── <main>            ← 핵심 본문 (딱 하나)
│    └── <section>    ← 주제별 덩어리
│         └── <article>  ← 그 자체로 완결된 글
├── <aside>           ← 본문 곁다리 (사이드바)
└── <footer>          ← 맨 아래 꼬리말
```

### section과 article, 뭐가 다르지?

가장 헷갈렸던 부분. 기준은 **"이것만 떼어내도 말이 되는가?"**

- **`<article>`**: 떼어내서 다른 곳에 옮겨도 그 자체로 완결된 콘텐츠. 블로그 글 하나, 뉴스 기사 하나, 상품 카드 하나.
- **`<section>`**: 혼자서는 완결되지 않고, 페이지 안에서 "주제별 묶음" 역할. 예: 내 소개 페이지의 "소개 / 스킬 / 연락처" 각 구역.

지금 쓰고 있는 이 블로그로 예를 들면, 글 하나하나가 `<article>`이고, "인기 글 목록" 같은 묶음이 `<section>`이다.

### div 대신 시맨틱을 쓰는 이유 (복습 + 확장)

어제 정리한 내용의 연장선. `<div>`로도 화면은 똑같이 나오지만:

개발자는 코드만 봐도 구조가 보인다. 검색엔진(SEO) 쪽에서는 어디가 핵심 콘텐츠인지 알아본다. 화면낭독기(접근성)를 쓰면 "메뉴 건너뛰고 본문으로" 같은 탐색도 가능해진다.

**결론: 뜻에 맞는 시맨틱 태그 먼저, 마땅한 게 없을 때만 div/span.**
