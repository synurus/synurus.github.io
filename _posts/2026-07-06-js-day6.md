웹개발 수업 6일차. HTML(뼈대), CSS(옷)에 이어 드디어 **JavaScript(움직임과 반응)** 시작. 버튼을 누르면 글자가 바뀌는 첫 코드를 작성했고, 변수·데이터 타입·연산자·조건문·함수까지 프로그래밍의 기초 재료를 배웠다. 심화에서는 Hello world와 변수 예제의 코드 패턴을 한 줄씩 해부했다.

## 1. 수업 내용 요약

### 1-1. JavaScript란?

| 기술 | 역할 |
|------|------|
| HTML | 뼈대 |
| CSS | 옷과 색깔 |
| **JavaScript** | **움직임과 반응** |

JS로 할 수 있는 것: 버튼 클릭하면 글자·그림 바꾸기, 사용자가 적은 이름 기억하기, 계산하고 결과 보여주기, 게임·애니메이션 만들기.

**수업 코드는 전부 ES6+(2015년 이후 최신 문법)로 작성한다:**
- 변수는 `let`, 상수는 `const` (옛날 `var` 안 씀)
- 함수는 화살표 함수 `() => { }`
- 문장 조립은 템플릿 리터럴 `` `${ }` `` (옛날 `+` 이어붙이기 대신)

### 1-2. Hello world! — 첫 JavaScript

버튼을 누르면 제목 글자가 바뀌는, 세 파일이 협동하는 가장 작은 예제.

```
02/
├─ index.html      # 화면 (h1 + 버튼)
├─ styles/main.css # 꾸미기
└─ scripts/main.js # 글자 바꾸기
```

```js
// 1. id가 "title"인 제목을 찾아 title 상자에 담기
const title = document.querySelector("#title");
// 2. id가 "btn"인 버튼을 찾아 btn 상자에 담기
const btn = document.querySelector("#btn");

// 3. btn을 클릭하면 안쪽 함수 실행
btn.addEventListener("click", () => {
  // 4. 제목 글자를 교체
  title.textContent = "Hello world!";
});
```

버튼을 눌렀더니 정말 글자가 바뀌었다. 자세한 해부는 아래 심화에서.

### 1-3. 변수 (Variables) — 값을 담는 상자

변수는 **이름표 붙은 상자**. 숫자·글자 같은 값을 넣어두고 이름만 부르면 안에 든 값이 따라온다.

```js
let myName = nameInput.value;          // 입력한 글자를 상자에 담기
out.textContent = `안녕, ${myName}!`;  // 템플릿 리터럴로 문장에 끼워 넣기
```

- `let` — "새 변수(상자) 만들게" 하는 마법 단어. 값을 나중에 바꿀 수 있음
- `const` — 한 번 담으면 고정되는 상자 (constant)
- `=` — "이 값을 상자에 넣어"라는 뜻. **같다는 비교가 아님!**

### 1-4. 데이터 타입 — 상자에 담는 값의 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| `String` | 글자 (따옴표로 감싸기) | `"Bob"` |
| `Number` | 숫자 (따옴표 없음) | `10` |
| `Boolean` | 참 또는 거짓 | `true`, `false` |
| `undefined` | 값을 아직 안 넣음 (선언만 한 변수) | `let x;` 한 뒤의 x |
| `null` | 의도적 "비어있음" 표시 | `null` |
| `Array` | 여러 값을 한 줄에 묶기 | `[1, "Bob", 10]` |
| `Object` | 거의 모든 것 (덩어리) | `document.querySelector("h1")` |

- `typeof 값` — 그 값의 타입을 글자로 알려준다 (`"string"`, `"number"` 등)
- `"3"`과 `3`은 다르다 — 따옴표가 있으면 글자!
- 재밌는 사실: `typeof null`은 `"object"`가 나온다. JS의 유명한 버그인데 하위 호환 때문에 못 고치고 그대로 남아있다고.

### 1-5. 연산자 & 주석

| 기호 | 역할 | 예시 |
|------|------|------|
| `+` | 더하기 / 글자 잇기 | `6 + 9` → 15 |
| `- * /` | 빼기·곱하기·나누기 | `8 * 2` → 16 |
| `=` | 상자에 값 넣기 (대입) | `let x = 3;` |
| `===` | 같은지 비교 (참/거짓) | `x === 3` → true |
| `!==` | 다른지 비교 | `x !== 5` → true |

- `+`는 숫자끼리는 더하기, **글자끼리는 이어 붙이기** — 그래서 입력값은 `Number(a.value)`로 숫자 변환부터
- `//` 뒤는 주석 — 컴퓨터는 무시, 사람만 읽는 메모 (HTML은 `<!-- -->`, CSS는 `/* */`, JS는 `//`)

### 1-6. 조건문 (if/else) — 만약 ~라면

"만약 이렇다면 이것을 하고, 아니면 저것을 해"라고 컴퓨터에게 시키는 갈림길.

```js
if (flavor.value === "chocolate") {
  result.textContent = "와! 초코 아이스크림 좋아! 🍫";
} else if (flavor.value === "vanilla") {
  result.textContent = "바닐라도 깔끔하니 좋지! 🍦";
} else {
  result.textContent = "음... 그래도 초코가 최고인데...";
}
```

- `if (조건)` — 괄호 안이 참(true)이면 바로 아래 중괄호 실행
- `else if` — 갈림길이 여러 개일 때 / `else` — 위 조건이 모두 거짓이면
- ⚡ `===` vs `==` — `===`는 **타입까지** 같아야 참. 실무는 항상 `===` 권장

### 1-7. 함수 (Functions) — 재사용하는 명령 묶음

자주 쓰는 명령을 하나로 묶어 이름 붙인 것. 자판기 버튼 하나로 음료가 나오듯, 이름만 부르면(호출) 묶어둔 일을 해준다.

```js
// num1, num2 = 매개변수 (받아올 재료)
const multiply = (num1, num2) => num1 * num2;

multiply(4, 7);   // 호출 — 답 28
```

- `const 이름 = (재료) => { }` — 화살표 함수 만들기
- **`=>` 뒤가 한 줄이면 `return`과 중괄호 생략** — `(a, b) => a * b`는 곱한 값을 자동으로 돌려준다
- 옛 표기 `function multiply() { return ... }`와 같은 동작, 화살표가 더 짧다

## 2. 어려웠던 것 — 심화 학습

첫 JS 코드가 짧아도 모르는 게 한가득이라, Hello world와 변수 예제를 한 줄씩 해부했다.

### 2-1. Hello world 코드 패턴 해부

**핵심 개념:** HTML은 구조, CSS는 스타일, JavaScript는 **상호작용**을 담당한다. 버튼을 클릭하면 제목 글자가 바뀌는 예제가 이 원리를 그대로 보여준다.

```js
const title = document.querySelector("#title");
const btn = document.querySelector("#btn");
btn.addEventListener("click", () => {
  title.textContent = "Hello world!";
});
```

**한 줄씩 뜯어보면:**

| 코드 | 정체 | 뜻 |
|------|------|-----|
| `const` | 키워드 | 변하지 않는 값을 담는 상자를 선언 |
| `document.querySelector()` | 메서드 | id로 페이지 요소를 찾기 |
| `() => { }` | 화살표 함수 | 이름 없는 함수를 만드는 최신 문법 |
| `addEventListener("click", 함수)` | 메서드 | 클릭 시 실행할 코드를 등록하는 이벤트 리스너 |
| `textContent` | 속성 | 요소의 텍스트를 읽거나 바꾸기 |

**속성 vs 메서드 구분법:** 점 뒤 이름에 괄호 `()`가 붙으면 메서드(동작), 안 붙으면 속성(값). `querySelector()`는 메서드, `textContent`는 속성.

**세 파일이 연결되는 원리 — 같은 id 하나로:**

```
HTML:  <h1 id="title">          ← id로 구조를 만들고
CSS:   #title { color: ... }    ← 같은 id 선택자로 스타일을 입히고
JS:    querySelector("#title")  ← 같은 id 문자열로 요소에 접근
```

**파일 연결 위치 — 왜 CSS는 위, JS는 아래?**

CSS는 `<head>` 안에, JavaScript는 `</body>` 직전에 두는 것이 성능과 요소 인식(로딩 순서) 측면에서 권장되는 방식이다.

- **CSS를 `<head>`에** — 브라우저가 화면을 그리기 전에 스타일을 알아야, 스타일 안 입은 화면이 번쩍이는 현상(FOUC)을 방지
- **JS를 `</body>` 앞에** — HTML을 다 그린 뒤 실행돼야 `querySelector`가 요소를 찾을 수 있다. 위에 두면 아직 요소가 없어서 `null`이 나온다!
- 참고: 강제 규칙은 아니고 관례. 실무에서는 `<head>`에 두고 `defer` 속성을 쓰는 방법도 권장된다 (다운로드는 병렬, 실행은 파싱 후, 순서 보장).

### 2-2. 변수 예제 해부 — 입력값을 담아 인사하기

이름을 입력하고 버튼을 누르면 입력값을 변수에 담아 화면에 인사말로 보여주는 예제.

```js
const nameInput = document.querySelector("#name");
const out = document.querySelector("#out");

document.querySelector("#greet").addEventListener("click", () => {
  let myName = nameInput.value;
  out.textContent = `안녕, ${myName}!`;
});
```

**주요 개념:**

- **`let` vs `const`** — `let`은 나중에 값이 바뀔 수 있는 상자, `const`는 값이 고정된 상자. 요소를 담을 땐 `const`(바뀔 일 없음), 사용자 입력처럼 매번 달라지는 값은 `let`.
- **`.value`** — 입력창(input)에 사용자가 적은 글자를 꺼내오는 속성. `textContent`가 일반 요소의 글자라면, `.value`는 입력칸 전용.
- **템플릿 리터럴 `` `${ }` ``** — 백틱(`` ` ``) 문자열 안에 변수 값을 끼워 넣는 최신 문법. 옛날 `"안녕, " + myName + "!"` 이어붙이기보다 훨씬 읽기 좋다. 백틱은 키보드 숫자 1 왼쪽 키 — 작은따옴표 `'`와 다르니 주의!
- **`=` vs `===`** — `=`는 값을 담는 **대입** 연산자, `===`는 값이 같은지 **비교**하는 연산자. 완전히 다른 일을 한다. 조건문에서 `=`를 쓰면 비교가 아니라 대입이 일어나버리는 고전적인 실수가 있다고.

**파일 구성 패턴:** HTML이 `input#name`, `button#greet`, `p#out`으로 구조를 만들면 → JS가 `querySelector`로 각 요소를 찾아, 버튼 클릭 시 입력값을 변수에 저장 후 문장으로 출력. **"HTML이 무대를 만들고 JS가 배우를 움직인다"**는 감각이 잡히기 시작했다.
