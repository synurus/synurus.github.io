## 1. 수업 내용 요약 ① — OpenAPI (공공 데이터)

### 1-0. 곁들여 배운 MCP — AI에게 능력을 더하는 USB

본격 API에 앞서 **MCP(Model Context Protocol)** 개념을 배웠다.

- **할루시네이션** — AI가 모르는 걸 아는 척 지어내는 것 ("세종대왕 맥북프로 던짐 사건"이 유명한 예)
- 해결책은 **AI를 진짜 도구에 연결**하는 것. 날씨를 물으면 기상청에 물어보게 만들기.
- 문제는 도구마다 연결 방식이 다르다는 것 → **MCP = "이렇게 연결하자"는 표준 약속.** 충전 케이블이 제각각이던 시절 → USB-C 하나로 통일된 것과 같은 구도.
- `fastmcp` 라이브러리를 쓰면 **파이썬 5줄로 나만의 MCP**도 만들 수 있다 (`@mcp.tool()` 데코레이터 + 함수 하나).

> 12일차에 Express 서버를 만들며 "AI에 도구를 연결한다"는 감각을 배웠는데, MCP는 그 연결 방식을 표준화한 것이었다.

### 1-1. ITS 공공 CCTV API — 인증키로 데이터 창고 열기

국토교통부 ITS(지능형교통시스템)에서 **전국 도로 CCTV 스트림 주소**를 받아왔다. OpenAPI = 공공기관 데이터 창고의 자동문, 인증키(apiKey)가 그 열쇠.

```python
url_cctv = (
    f"https://openapi.its.go.kr:9443/cctvInfo"
    f"?apiKey={key}&type=its&cctvType=1"
    f"&minX={minX}&maxX={maxX}&minY={minY}&maxY={maxY}&getType=json"
)
response = urllib.request.urlopen(url_cctv)             # ① 요청 → 응답 봉투
json_str = response.read().decode("utf-8")              # ② bytes → str (사람 글자)
json_object = json.loads(json_str)                      # ③ str → dict (사전)
cctv_play = pd.json_normalize(json_object["response"]["data"], sep='')  # ④ 표로 펴기
test_url = cctv_play["cctvurl"][77]                     # ⑤ 77번 CCTV 주소
```

**변환이 두 번 필요한 이유가 핵심이었다:** 서버는 전송용 bytes로 보내니 `decode`로 글자를 만들고, 글자 덩어리에서 값을 꺼내려면 `json.loads`로 사전(dict)으로 바꿔야 한다. 그리고 중첩이 깊은 dict은 `pd.json_normalize`로 **엑셀 같은 평평한 표(DataFrame)**로 펴서 다룬다 — pandas 첫 사용.

- `minX/maxX/minY/maxY` = 지도에 사각형을 그려 그 안의 CCTV만 요청하는 좌표 범위
- 쿼리스트링 `?key=value&...` 구조는 12일차 날씨 API 때 배운 그대로

### 1-2. CCTV + YOLO 결합 — 지난주가 여기서 만난다

받아온 URL을 `cv2.VideoCapture()`에 그대로 넣으면 끝. 16일차에 m3u8 스트림으로 하던 걸, 오늘은 **API로 주소를 동적으로 받아와서** 한다.

```python
test_url = cctv_play["cctvurl"][1000]
cap = cv2.VideoCapture(test_url)     # URL도 웹캠처럼
model = YOLO("yolo11n.pt")
while cap.isOpened():
    success, frame = cap.read()
    if not success: break
    cv2.imshow("OPENAPI", model(frame)[0].plot())
    if cv2.waitKey(1) & 0xFF == ord("q"): break
cap.release(); cv2.destroyAllWindows()
```

### 2-2. LLM 채팅 — ChatGPT와 같은 인터페이스

```python
completion = client.chat.completions.create(
```

## 3. 수업 내용 요약 ③ — Transformers (문장을 이해하는 AI)

`pipeline` 하나로 태스크 이름만 바꾸면 모델·토크나이저가 자동 결합되는 게 핵심. **YOLO에서 모델명만 바꿔 기능이 바뀌던 것과 같은 설계 철학**이다.

### 3-1. 문장 유사도 — 문장을 좌표로 바꾸기

```python
from sentence_transformers import SentenceTransformer, util
model = SentenceTransformer("all-MiniLM-L6-v2")   # 384차원 벡터 출력

emb1 = model.encode(sen1, convert_to_tensor=True)
emb2 = model.encode(sen2, convert_to_tensor=True)
cos_sim = util.pytorch_cos_sim(emb1, emb2)
print(f"유사도 : {cos_sim.item():.4f}")
```

- **임베딩** = 문장을 숫자 좌표(384차원 점)로 바꾸는 것. 의미가 비슷하면 좌표도 가깝다.
- **코사인 유사도** = 두 화살표의 **각도만** 비교 (길이 무시 → 문장 길이 영향 없음). 1=동일, 0=무관, -1=반대.
- 어순만 다른 문장("He is reading a book in the library" ↔ "He is at the library reading a book") → **0.94**, 완전히 다른 문장 → **0.12**. 숫자로 확인하니 신기했다.

용어도 한 세트 정리했다: 임베딩 · 벡터 · 코퍼스(AI가 공부한 책더미) · 토크나이즈(문장 쪼개기) · 트랜스포머(2017 구글, 단어끼리 누굴 가리키는지 한꺼번에 파악) · 텐서(벡터의 N차원 확장, **벡터 = 1차원 텐서**).

### 3-2~3-5. pipeline 4종

| 태스크 | 코드 | 결과 · 한계 |
|--------|------|------------|
| **감정 분석** | `pipeline("sentiment-analysis")` | `{'label': 'POSITIVE', 'score': 0.9998}` — score는 softmax 확률(확신도). 영어 전용, 중립 미지원 |
| **텍스트 생성** | `pipeline("text-generation", model="gpt2")` | 시드 문장 뒤를 이어 씀. `max_new_tokens` 없으면 무한 생성 위험 |
| **영문 요약** | `pipeline("summarization", model="t5-small")` | `min_length`/`max_length`로 길이 제어, `do_sample=False`면 매번 같은 결과 |
| **요약+번역** | `+ GoogleTranslator(source='en', target='ko')` | 영어 모델의 한계를 외부 번역기로 보완 |

**T5의 Task Prefix**가 인상적이었다 — 같은 모델에 `"summarize: "`, `"translate English to Korean: "`처럼 **머리말만 바꿔** 다른 일을 시키는 구조. `pipeline("summarization")`을 쓰면 라이브러리가 접두사를 자동으로 붙여준다. 그리고 **인코더**(글자→숫자, 원문 이해)와 **디코더**(숫자→글자, 문장 생성)의 역할 분담도 여기서 정리됐다.

마지막 예제는 **파이프라인 결합**이었다: 영어 원문 → T5 요약(영) → GoogleTranslator → 요약(한). 도구 하나가 못 하는 일을 **여러 도구를 이어 붙여** 해결하는 방식.

## 5. 어려웠던 것 — 심화 학습

오늘은 공교롭게도 **막힌 상황이 두 번** 있었고, 둘 다 "다른 도구로 갈아타기"로 풀렸다.

### 5-1. HuggingFace 토큰 부족 → Pollinations.ai로 갈아타기

**문제:** STEP 03(이미지 생성)에서 무료 토큰 한도가 너무 적어 FLUX.1-dev 호출을 제대로 진행할 수 없었다. 이미지 생성은 텍스트 생성보다 연산이 훨씬 무거워서 무료 할당량이 금방 바닥난다.

**대안:** AI에게 물어보니 **[Pollinations.ai](https://pollinations.ai)**를 추천해줘서 그쪽으로 진행했다.

> Pollinations.ai는 **별도의 회원가입이나 API 키 발급 없이 완전히 무료로 사용할 수 있는 이미지 생성 서비스**다. **HTTP GET 요청(URL 방식)**으로 호출할 수 있어서 파이썬 코드 몇 줄만으로 즉시 이미지를 만들어낼 수 있다.

핵심 차이가 재밌다 — HuggingFace는 **SDK를 설치하고 키로 인증**해야 하는데, Pollinations는 **주소를 부르는 것만으로** 이미지가 온다. 12일차에 배운 "택배 보내기 3칸(주소·인증·내용)" 중 인증 칸이 통째로 사라진 셈. 별도 라이브러리 없이 `requests`나 `urllib`로 바로 받을 수 있다.

**배운 점:** 무료 서비스의 할당량은 실습의 상수(常數)가 아니다. 하나가 막혔을 때 **같은 일을 하는 다른 통로**를 찾는 것도 실력이고, 그래서 오늘 배운 것 중 "API는 인증 방식만 다를 뿐 뼈대는 같다"는 감각이 큰 도움이 됐다.

### 5-2. Tesseract가 한글을 못 읽는다 → EasyOCR로 갈아타기 ★

**문제:** STEP 03(lang 옵션)에서 한글이 담긴 책 표지 사진을 `lang='kor'`로 분석했는데, 결과가 처참했다.

**Tesseract 결과:**

```
이 59 8 두 도 두 89 = = 0
| 베기 |
거고 그기 |) |! |
"0   |           | 0 으으 을 을 |
"배 |     「    | |   |  띠 0) 000표 0 |패기 0 |
띠 띄 00 100 |
8060ㅁ566000006ㅁ
```

글자라고 부를 수 있는 게 거의 없다. 원본 사진에는 제목("먼저 온 미래"), 세로로 회전된 부제, 추천사, 출판사명이 있었는데 **하나도 못 잡았다.**

**대안 — EasyOCR.** AI가 추천해줘서 사용해봤는데 **상태가 훨씬 나았다.**

**EasyOCR 결과 (일부):**

```
먼저 온
미래
...
"우리논 늘 미래가 멀리 있다고 착각하다.
...
정재승
(KAIST 교수, ...)
동아시야
추천
```

제목·추천사·인물명·출판사까지 **읽을 수 있는 문장으로** 나왔다. 오타가 좀 있지만("우리논", "동아시야") 원문을 알아볼 수 있는 수준.

**왜 이렇게 차이가 날까 — 찾아본 이유:**

| | Tesseract | EasyOCR |
|---|-----------|---------|
| 방식 | 전통적 패턴 매칭 기반 (+LSTM) | **딥러닝 기반** (탐지 + 인식 2단계) |
| 강점 | 깨끗한 스캔 문서, 영어 | 사진·간판·복잡한 배경, 다국어 |
| 한글 | 언어팩 있어도 품질 편차 큼 | 상대적으로 안정적 |
| 기울어진/회전된 글자 | 취약 | 탐지 단계에서 영역을 먼저 찾아 상대적으로 강함 |

이번 이미지가 특히 어려운 조건이었다 — **배경이 빌딩 사진**(글자 아닌 격자무늬가 가득), **글자가 흰 박스 위에 얹혀 있고**, 일부는 **세로로 90도 회전**되어 있었다. 수업 페이지에도 나왔듯 "사진 해상도가 낮거나 글꼴이 특수하면 인식률 저하"인데, 여기에 회전·복잡한 배경까지 겹친 것. Tesseract는 빌딩 창문 격자까지 글자로 착각해 `0`, `ㅁ`, `|` 같은 기호를 쏟아낸 것으로 보인다.

**교훈 3가지:**

1. **도구에는 적성이 있다** — Tesseract가 나쁜 게 아니라, "깨끗한 문서 스캔"이 그의 무대다. 사진 속 글자는 EasyOCR/PaddleOCR 같은 딥러닝 OCR의 영역. 수업 페이지도 이 둘을 "더 똑똑한 대안"으로 소개하고 있었다.
2. **전처리도 방법이었다** — 도구를 바꾸기 전에 흑백 변환·크기 조정·회전 보정 같은 전처리를 시도해볼 수도 있었다. 15일차에 배운 데이터 증강(PIL)의 기술이 여기서도 쓰인다.
3. **결과를 눈으로 검증하는 습관** — OCR은 "돌아갔다 = 성공"이 아니다. 출력이 쓰레기여도 프로그램은 에러 없이 끝난다. 13일차 심화(AI 출력을 정규식으로 검증)와 같은 이야기 — **AI의 출력은 늘 검증 대상**이다.
