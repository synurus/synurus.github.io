## 1. 수업 내용 요약

### 1-1. React Router — 여러 페이지 만들기

React는 기본이 한 페이지(SPA). 주소별로 화면(홈·소개·상세)을 나누려면 라우터 라이브러리를 설치한다 — 실무 거의 필수.

```bash
npm install react-router-dom
```

```jsx
// App.jsx — 주소 ↔ 화면 연결
import { BrowserRouter, Routes, Route } from 'react-router-dom'

<BrowserRouter>
  <Routes>
    <Route path="/" element={<Home />} />
    <Route path="/about" element={<About />} />
  </Routes>
</BrowserRouter>
```

```jsx
// 페이지 이동 두 가지
<Link to="/about">소개 보기</Link>                          {/* ① 링크 (권장) */}
<button onClick={() => navigate('/about')}>이동</button>    {/* ② useNavigate — 코드로 이동 */}
```

- `path`(주소) ↔ `element`(보여줄 화면)를 짝지음
- **새로고침 없이** 화면만 빠르게 전환 → 앱처럼 매끄러움
- 페이지는 `src/pages/`에 모으는 게 실무 정석 (재사용 부품은 `components/`)

### 1-2. 흔한 실수 & 꿀팁 정리 ★

화면이 안 바뀌거나 에러 날 때 여기부터 확인하라는 체크리스트. 특히 와닿은 것들:

| 실수 | 무슨 일이 | 이렇게 |
|------|----------|--------|
| `{0 && <X/>}` | 화면에 **"0"이 찍힘** | `{count > 0 && <X/>}` |
| `onClick={fn()}` | 그릴 때 **바로 실행**됨 | `onClick={fn}`, 인자 땐 `{() => fn(x)}` |
| `arr.push(x)`로 state 변경 | 화면 안 바뀜 (같은 배열이라) | `setArr([...arr, x])` **새 배열로** |
| `setCount(count+1)` 여러 번 | 한 번만 적용됨 | `setCount(c => c + 1)` **함수형** |
| fetch를 컴포넌트 본문에 바로 | 그릴 때마다 호출 (무한) | `useEffect(..., [])` 안에서 |

지난 글에 심어뒀던 질문의 답을 찾았다 — **`setCount(c => c + 1)` 함수형 표기는 연속 호출에도 안전**하기 때문. `setCount(count + 1)`을 두 번 불러도 둘 다 같은 count를 보고 있어서 한 번만 오르지만, 함수형은 "직전 값 받아서 +1"이라 정확하다.

**화면이 안 바뀔 때 점검 순서:** ① set 함수로 바꿨나 → ② 배열·객체는 새로 만들었나(`[...arr]`) → ③ F12 콘솔의 빨간 에러부터 읽기.

## 2. 미니 프로젝트 — K-POP 인기 차트 앱 🎵

수업 42번의 기본 주제(React 용어 사전)를 바꿔서, **Apple Music 공식 차트 피드로 실시간 인기곡을 보여주는 앱**을 만들었다. 기획서(PLANNING.md) 기준 간단 요약.

### 무엇을 만들었나

| 항목 | 내용 |
|------|------|
| 앱 | K-POP 차트 (Vite + React + React Router) |
| 데이터 | Apple Music 공식 차트 피드 — **회원가입·API 키 불필요**, 한글 그대로, 실시간 |
| 홈 | 🇰🇷K-POP ↔ 🌍Global 전환, 곡/앨범/신곡 탭, TOP10/25/50, 검색 필터, NEW 뱃지, 로딩 스켈레톤 |
| 상세 | `useParams()`로 곡 찾아 표시 (600px 고화질 커버, 순위, Apple Music 링크) |
| 추가 | ♥ 즐겨찾기(localStorage), 다크 모드, 404 페이지, Netlify 배포 대응 |

수업에서 배운 게 전부 실전 투입됐다 — props(TrackCard), map+key(카드 목록), 조건부 렌더링(로딩/에러/NEW 뱃지), useState(탭·검색어 등 7개), controlled input(검색창), **useEffect `[country, chartType, limit]`**(탭 바뀔 때마다 다시 fetch), Router(`Link`·`useParams`), 그리고 JS 수업의 filter/sort/localStorage까지.

### 가장 기억에 남는 문제 3가지

**① CORS — 브라우저가 fetch를 거부한다.** Apple 피드가 CORS를 허용하지 않아 브라우저에서 직접 못 가져온다. → **Vite 프록시**로 해결: 내 개발 서버가 대신 Apple에 다녀오는 구조 (브라우저 → 내 서버는 같은 출처라 OK). 배포 후엔 프록시가 없어 다시 겪었고, Netlify의 `_redirects` 규칙으로 재해결. **프록시는 개발용이라는 걸 몸으로 배움.**

**② 상세 갔다 오면 탭이 리셋된다.** `navigate(-1)`로 돌아와도 페이지 컴포넌트가 새로 만들어지며 state가 초기화됐다. → 차트 선택 상태를 안 사라지는 부모(App)로 **상태 끌어올리기**. "라우트가 바뀌면 페이지의 state는 죽는다"는 걸 이해한 순간.

**③ K-POP만 골라내기.** 피드에 국적 정보가 없어서, 장르가 K-Pop이거나 **가수명·제목에 한글이 있으면**(`/[가-힣]/`) 통과시키는 근사 필터를 만들었다. 공개 API가 원하는 모양 그대로 주는 경우는 드물다 — **가공이 실무**다.

### 그 외 배운 점 (기획서에서 발췌)

- Grid `1fr` 함정: 긴 제목이 칸을 밀어냄 → 말줄임엔 `minmax(0, 1fr)` 필수
- localStorage + state 정석: 시작할 때 `useState(() => 저장값 읽기)`, 바뀔 때 `useEffect`로 저장
- 다크 모드 = CSS 변수 값 교체 몇 줄 (색을 `:root`에 모아둔 덕) — HTML/CSS 프로젝트 때와 같은 원리!
- fetch cleanup: 탭 연타 시 이전 요청이 새 화면을 덮어쓰는 경합을 `ignore` 표시로 방지 (보충 학습한 cleanup을 바로 실전에!)
- SPA 배포와 새로고침: `/favorites`에서 새로고침하면 서버엔 그 파일이 없어 404 → `/* /index.html 200` 규칙 필요
