웹개발 수업 4일차. CSS의 핵심 개념들을 한꺼번에 훑었다 — 박스 모델, flex, 가상클래스, 애니메이션, position, Grid까지. 화면이 드디어 "움직이기" 시작한 날. 심화에서는 `rem` 단위의 정체와 CSS 쉼표의 함정, 그리고 Claude Code로 반복 작업을 자동화한 경험을 정리했다.

## 1. 수업 내용 요약

### 1-1. Stylelint — CSS 자동 검사·정렬 도구

CSS 전용 린터(검사기). 오타(`colur`), 세미콜론 누락, 중복 선택자, 속성 순서를 사람 대신 잡아준다. 지난 시간에 배운 "일반 → 구체 순서는 외울 필요 없다"의 답이 이것.

```bash
# 설치 (한 번만) — Node.js 먼저 필요 (npm은 Node에 포함)
npm install -D stylelint stylelint-config-standard stylelint-order

# 검사
npx stylelint "styles/main.css"

# 자동 수정 (정렬·오류)
npx stylelint "styles/main.css" --fix
```

설정은 `.stylelintrc.json` 파일로:

```json
{
  "extends": ["stylelint-config-standard"],
  "plugins": ["stylelint-order"],
  "rules": { "order/properties-alphabetical-order": true }
}
```

- `--fix` 한 번이면 속성이 알파벳순으로 자동 정렬된다. 순서만 바뀌고 화면 결과는 동일.
- ⚠️ `npm init stylelint`은 쓰지 말 것 — 대화형 마법사가 실패하기 쉽고 설정 파일이 겹친다.
- JSON 파일에는 주석을 넣으면 안 됨 (문법 깨짐).
- Stylelint는 CSS만 담당. HTML 검사는 **HTMLHint**로 따로 (`npx htmlhint "index.html"`).

### 1-2. 박스 모델 (padding · margin · border)

**모든 요소는 네모 상자**다. 액자에 비유하면:

| 속성 | 뜻 | 액자 비유 |
|------|-----|----------|
| `padding` | 상자 **안쪽** 여백 | 액자와 사진 사이 |
| `border` | 상자 테두리 | 액자 틀 |
| `margin` | 상자 **바깥쪽** 여백 | 옆 액자와의 거리 |

```css
ul {
  padding: 20px;              /* 안쪽 여백 */
  border: 2px solid navy;     /* 두께 · 종류 · 색을 한 줄에 */
  margin: 20px;               /* 바깥 여백 */
}
```

**값 개수로 방향 지정** (padding도 규칙 동일):

```css
margin: 20px;                  /* 1개 = 상하좌우 전부 */
margin: 10px 30px;             /* 2개 = 상하 / 좌우 */
margin: 10px 20px 30px 40px;   /* 4개 = 상→우→하→좌 (12시부터 시계방향) */
margin: 0 auto;                /* 상하 0, 좌우 auto = 가로 가운데 정렬 (width 필요) */
```

### 1-3. 가로로 나란히 — flex (맛보기)

상자는 기본적으로 위→아래로 쌓인다. 가로로 두려면 **부모에** `display: flex` 한 줄.

```css
.menu {
  display: flex;      /* 자식을 가로로 나란히 */
  gap: 10px;          /* 자식 사이 간격 */
  list-style: none;   /* 목록 점(•) 제거 */
}
```

부모에 적용하면 자식들이 나란히 선다. 본격 레이아웃은 나중에 — 지금은 **"가로 배치는 flex"** 정도만.

### 1-4. CSS 붙이는 3가지 방법 (외부 · 내부 · 인라인)

| 방법 | 위치 | 적용 범위 |
|------|------|----------|
| **외부** `<link>` | 별도 `main.css` 파일 | 여러 페이지 공유 — **권장** |
| 내부 `<style>` | `<head>` 안 | 한 페이지 |
| 인라인 `style=""` | 태그에 직접 | 한 요소 |

충돌 시 우선순위: **인라인 > 내부·외부**. 관리 편의성 때문에 보통 외부 방식을 쓴다.

### 1-5. 상태에 따라 — 가상클래스 (:hover 등)

선택자 뒤에 콜론 `:` + 상태 = 가상클래스. "그 순간"에만 적용된다.

```css
a:link    { color: royalblue; }  /* 방문 전 */
a:visited { color: purple; }     /* 방문 후 */
a:hover   { color: tomato; }     /* 마우스 올림 */
a:active  { color: red; }        /* 누르는 중 */
```

⚠️ **순서 필수 — LVHA ("러브하", LoVe-HAte)**: `:link` → `:visited` → `:hover` → `:active`. CSS는 같은 조건이면 아래 규칙이 이기기 때문에, `:hover`를 `:visited`보다 위에 쓰면 방문한 링크에서 hover 효과가 덮여 안 보인다.

### 1-6. transform + transition — 움직이는 효과

**hover 회전 + 확대:**

```css
.box {
  transition: transform 0.3s ease;  /* 바뀔 때 0.3초에 걸쳐 부드럽게 */
}
.box:hover {
  transform: rotate(10deg) scale(1.2);  /* 10도 기울이고 1.2배 확대 */
}
```

- `transform` = 모양 변형 (`rotate` 회전, `scale` 확대, `translateY` 이동)
- `transition` = 변형에 시간을 줘서 뚝딱이 아니라 **스르륵**
- 끝의 easing = 속도 리듬: `ease`(느→빠→느, 기본), `linear`(일정, 로딩바), `ease-out`(끝에 느리게) 등

**카드 뒤집기 (3D flip)** — 응용판:

```css
.card { perspective: 600px; }                       /* 보는 거리 — 작을수록 더 휘어 보임 */
.inner {
  transition: transform 0.6s;
  transform-style: preserve-3d;                     /* 자식을 3D 공간에 (없으면 납작) */
}
.card:hover .inner { transform: rotateY(180deg); }  /* Y축 180도 = 뒤집기 */
.front, .back {
  position: absolute; inset: 0;                     /* 앞뒷면을 겹쳐 쌓기 */
  backface-visibility: hidden;                      /* 뒤로 돌아간 면은 숨김 */
}
.back { transform: rotateY(180deg); }               /* 뒷면은 미리 돌려둠 */
```

### 1-7. 자동 애니메이션 (@keyframes)

`:hover` 없이도 저절로 움직이게 하려면:

```css
@keyframes float {                       /* 움직임 "대본" 정의 */
  0%, 100% { transform: translateY(0); }      /* 시작·끝: 제자리 */
  50%      { transform: translateY(-30px); }  /* 중간: 위로 30px (음수 = 위) */
}
.ball {
  animation: float 1.5s ease-in-out infinite;  /* 대본을 1.5초씩 무한 반복 */
}
```

- `@keyframes 이름` — 0%→50%→100% 순간의 모습을 정의
- `animation` — 그 대본을 요소에 적용 (시간·리듬·반복). `infinite` = 무한 반복

### 1-8. 네온 그라디언트 카드

```css
.neon {
  background: linear-gradient(135deg, #ec4899, #8b5cf6);  /* 135도 방향 분홍→보라 */
  box-shadow: 0 0 25px rgba(236, 72, 153, 0.7);           /* 흐림 반경 크게 = 네온빛 */
}
```

`box-shadow`의 큰 흐림 반경(25px)이 네온처럼 번지는 빛을 만든다. `rgba` 마지막 값은 투명도.

### 1-9. 요소 위치 잡기 — position

기본값 `static`(원래 흐름대로) + 활용 4종. 짝꿍 `top·right·bottom·left`로 미세 위치 지정.

| 값 | 기준 | 특징 |
|-----|------|------|
| `relative` | 자기 원래 자리 | 살짝 이동, 자리는 그대로 차지 |
| `absolute` | 가장 가까운 position 지정 조상 | 흐름에서 빠짐 (자리 안 차지) |
| `fixed` | 화면(뷰포트) | 스크롤해도 그 자리 |
| `sticky` | 스크롤 지점 | 스크롤하다 일정 지점에서 착 붙음 |

**실무 패턴이 핵심:** 4종을 외우는 게 아니라 "겹치기·고정"이 필요할 때 패턴으로 고른다.

- 카드 우상단 `NEW` 뱃지, 툴팁 → **부모 `relative` + 자식 `absolute`** 짝꿍 (가장 흔함)
- 떠있는 채팅 버튼, 상단 고정 메뉴바 → `fixed`
- 스크롤하다 붙는 테이블 헤더, 사이드 목차 → `sticky`
- 겹칠 땐 `z-index` 숫자 큰 쪽이 위

평소엔 `static`(기본)이 90%라고 한다.

### 1-10. 2차원 격자 — Grid

flex는 가로 한 줄, **grid는 행+열 동시**. 카드·갤러리처럼 반듯한 격자에 딱.

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3칸 균등 (1fr 1fr 1fr과 같음) */
  gap: 12px;                              /* 칸 사이 간격 */
}
```

자식을 안 건드려도 부모 한 곳만으로 카드 6개가 3개씩 두 줄로 자동 배치된다.

**`fr`이 뭐지?** fraction(분수) — 남은 공간을 나눠 갖는 **비율 단위**.
- `1fr 2fr 1fr` = 1:2:1 비율
- `200px 1fr` = 사이드바 고정 + 본문 가변
- `%` 대신 `fr`을 쓰는 이유: `%`는 gap까지 계산에 넣어 합이 넘쳐 줄바꿈이 꼬이지만, `fr`은 gap을 먼저 뺀 뒤 남은 폭만 나눠서 깔끔하다.

## 2. 어려웠던 것 — 심화 학습

### 2-1. `border: 1rem solid;`의 rem이 뭐야?

수업 코드에 나온 `rem`이 궁금해서 파봤다.

**`rem` = root em.** `html` 요소(루트)의 `font-size`를 기준으로 하는 상대 단위다. 브라우저 기본 폰트 크기가 16px이므로, 별도 설정이 없다면:

```css
border: 1rem solid;   /* = border: 16px solid; */
```

**그럼 `em`은 뭐의 약자?** 인쇄에서 유래한 단위로, 활판 인쇄 시절 대문자 **M**의 너비가 그 글꼴의 기준 크기와 비슷했던 데서 이름이 왔다는 설이 일반적이다 (정확한 어원은 다소 불명확).

**em vs rem — 핵심 차이:**

| 단위 | 기준 | 중첩 시 |
|------|------|---------|
| `em` | 가장 가까운 상위(또는 자신)의 font-size | 값이 **누적**되어 예측이 어려울 수 있음 |
| `rem` | 무조건 `html`(root)의 font-size 하나 | 항상 **일관된** 계산 |

`rem`의 진짜 강점: `html { font-size: 20px; }` 한 줄만 바꾸면 `1rem = 20px`이 되어 **페이지 전체 크기를 한 번에 조정**할 수 있다. 지난주 배운 CSS 변수처럼 "한 곳만 고치면 전체 반영"되는 구조.

### 2-2. CSS에서 쉼표(,)의 중요성

카드 뒤집기 코드에서 제대로 헷갈렸던 부분:

```css
.card:hover .inner { ... }   /* 쉼표 없음 */
.front, .back { ... }        /* 쉼표 있음 */
```

`.front, .back`에서 쉼표를 빼먹으면 작동이 안 됐다. 바로 위 줄엔 쉼표가 없는데 왜? Claude Code에 질문해서 정리했다 (답변은 `./03_CSS/08/CSS_선택자와_쉼표.md`로 저장).

| 구분자 | 뜻 | 예시 | 해석 |
|--------|-----|------|------|
| **쉼표 `,`** | "그리고 또" — 그룹 선택자 | `.front, .back` | .front**와** .back **둘 다**에 같은 규칙 |
| **공백 ` `** | "~안의" — 자손 선택자 | `.card:hover .inner` | hover된 .card **안의** .inner |

그러니까 `.front, .back`에서 쉼표를 빼고 `.front .back`이라고 쓰면 뜻이 완전히 달라진다 — **".front 안에 있는 .back"**을 찾게 되는데 그런 구조가 없으니 아무것도 선택되지 않아 작동이 안 됐던 것. 문법 오류가 아니라 **멀쩡히 동작하는 다른 뜻의 코드**가 돼버려서 에러도 안 뜨고 조용히 실패한다. 이게 더 무섭다.
