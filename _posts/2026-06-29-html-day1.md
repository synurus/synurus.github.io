웹개발 수업 첫날. HTML이 무엇인지부터 시작해서, 개발환경을 세팅하고 마지막에는 배운 태그를 전부 합쳐 간단한 자기소개 페이지까지 만들어봤다.

## 1. 수업 내용 요약

### 1-1. 큰 그림 — 웹페이지는 3가지로 만들어진다

본격적으로 들어가기 전에, 웹페이지 하나가 어떻게 구성되는지부터 배웠다. 레고로 집을 만드는 것에 비유하면 이해가 쉽다.

| 기술 | 역할 | 레고 비유 |
|------|------|-----------|
| **HTML** | 무엇이 있는지 (글·사진·버튼) | 집의 뼈대 |
| **CSS** | 어떻게 보이는지 (색·크기·배치) | 색칠·꾸미기 |
| **JS** | 어떻게 움직이는지 (클릭·반응) | 전기·스위치 |

이번 과목에서는 뼈대인 **HTML**부터 배운다.

### 1-2. 개발환경 세팅 (VSCode + Live Server)

코드는 **VSCode**로 작성한다. 메모장의 똑똑한 버전으로, 코드에 색이 입혀져서 구조가 잘 보인다.

**작업 폴더 만들기**
1. 바탕화면에 새 폴더 생성 → 이름 `01_HTML`
2. VSCode에서 `File → Open Folder` → `01_HTML` 선택
3. 안내 창이 뜨면 "Yes, I trust the authors" 클릭

**Live Server 설치 (최초 1회)**
- 왼쪽 확장(Extensions) 아이콘 → `Live Server` 검색 → Install
- 저장(`Ctrl+S`)할 때마다 브라우저가 자동 새로고침돼서 결과를 바로 확인할 수 있다.

`index.html` 파일을 만들고 오른쪽 아래 **Go Live**를 누르면 브라우저에 내 첫 페이지가 뜬다. 내가 쓴 글을 브라우저가 읽어서 화면에 그려준 것!

> 💡 **트러블슈팅**
> - Go Live 버튼이 안 보임 → Live Server 설치 여부 확인
> - 한글이 `???`로 깨짐 → `<meta charset="UTF-8">` 있는지 확인

### 1-3. 주석과 VSCode 단축키

**주석**은 화면에 보이지 않는 메모다. 브라우저가 무시하기 때문에, 미래의 나(또는 팀원)에게 설명을 남기거나 코드를 지우지 않고 잠깐 꺼둘 때 쓴다.

```html
<!-- 이건 메모. 화면에 안 보임 -->
<h1>안녕하세요</h1>
<!-- <p>이 줄은 잠깐 꺼둔 것</p> -->
```

**자주 쓰는 단축키** (Windows 기준, Mac은 Ctrl → Cmd)

| 단축키 | 기능 |
|--------|------|
| `Ctrl + /` | 주석 켜고 끄기 |
| `Ctrl + S` | 저장 |
| `Shift + Alt + ↓` | 줄 복제 |
| `Ctrl + F` | 찾기 |

### 1-4. 태그란? + 웹페이지의 뼈대

HTML은 글마다 **태그(tag)**라는 이름표를 붙인다. "이건 큰제목이야"라고 알려주는 표시다.

```html
<h1>안녕</h1>
```

- `<h1>` : 여는 태그 — "여기부터 큰제목"
- `안녕` : 내용
- `</h1>` : 닫는 태그 — `/`는 "여기서 끝"이라는 뜻

이 셋을 합쳐 **요소(element)**라고 부른다. 태그는 거의 항상 열기+닫기 한 쌍이고, 닫기를 빠뜨리면 화면이 깨진다.

**모든 HTML 문서가 공유하는 뼈대:**

```html
<!DOCTYPE html>          <!-- "이건 HTML 문서야" 선언 -->
<html lang="ko">         <!-- 문서 시작, 한국어 페이지 -->
<head>                   <!-- 화면엔 안 보이는 준비 칸 -->
  <meta charset="UTF-8"> <!-- 한글 안 깨지게 -->
  <title>제목</title>    <!-- 브라우저 탭에 뜨는 이름 -->
</head>
<body>                   <!-- 화면에 보이는 진짜 내용 칸 -->
  <h1>안녕하세요</h1>
</body>
</html>
```

**핵심:** 눈에 보이는 건 전부 `<body>` 안에. `<head>`는 무대 뒤 준비 공간이다.

### 1-5. 제목과 문단 (h1~h6, p)

- 제목은 `<h1>`(가장 큼) ~ `<h6>`(가장 작음)까지 6단계. 숫자가 커질수록 글씨가 작아진다.
- `<p>`는 글 한 덩어리(문단).

```html
<h1>가장 큰 제목</h1>
<h2>중간 제목</h2>
<p>저는 코딩을 처음 배우는 학생입니다</p>
```

신문에 비유하면 `<h1>`은 1면 헤드라인, `<p>`는 본문이다.

### 1-6. 강조와 태그 중첩 (strong, nesting)

태그 안에 태그를 넣는 것을 **중첩(nesting)**이라 한다. 선물 상자 안에 작은 상자를 넣는 느낌.

```html
<p>저는 <strong>정말</strong> 신납니다</p>  <!-- '정말'만 굵게 -->
```

**중요 규칙:** 먼저 연 태그를 나중에 닫는다. 바깥 상자를 마지막에 닫듯이. 순서가 엇갈리면(`<p><strong>…</p></strong>` ❌) 화면이 깨진다.

**특수문자(엔티티):** `<`, `>`, `&`는 브라우저가 명령어로 착각한다. 글자 그대로 보여주려면 약속된 기호로 바꿔 쓴다.
- `&lt;` → `<` / `&gt;` → `>` / `&amp;` → `&`

### 1-7. 이미지 (img)

```html
<img src="https://picsum.photos/300/200" alt="예시 사진" />
```

- `src` : 사진 주소
- `alt` : 사진이 안 뜰 때 대신 보일 글자

여기서 **속성(attribute)** 개념도 배웠다. 태그에 추가 정보를 `이름="값"` 형태로 적는 것. 태그 이름 뒤에 한 칸 띄고, 값은 따옴표로 감싼다.

`<img>`는 넣을 내용이 없어서 닫는 태그가 없는 **빈 요소(void element)**다. `<br>`, `<hr>`, `<meta>`도 같은 부류.

### 1-8. 링크 (a, href)

```html
<a href="https://www.google.com">구글로 가기</a>
```

- `<a>` : 누르면 다른 곳으로 이동하는 문
- `href` : 이동할 주소 — **`https://`까지 적어야 동작한다!**

### 1-9. 목록과 시맨틱 태그 (ul·li, header·main·footer)

```html
<header>맨 위 (로고·제목)</header>
<main>
  <ul>
    <li>좋아하는 것: 영화</li>
    <li>취미: 산책</li>
  </ul>
</main>
<footer>맨 아래 (저작권·연락처)</footer>
```

- `<ul>` = 점(•) 찍힌 목록 상자, `<li>` = 목록 안의 한 줄
- `<header>` / `<main>` / `<footer>` = 페이지를 위→가운데→아래로 나누는 **뜻이 담긴 이름표 상자(시맨틱 태그)**

암기 포인트: **"위는 header, 본문은 main, 아래는 footer"** — `<main>`은 한 페이지에 딱 하나만.

### 1-10. 합치기 — 자기소개 페이지 (1차)

오늘 배운 걸 전부 합쳐 자기소개 페이지를 만들었다. 구조는 이렇다:

```html
<body>
  <header>
    <h1>OO의 소개 페이지</h1>
  </header>
  <main>
    <img src="..." alt="내 사진" />
    <p>안녕하세요. 저는 웹 개발을 배우는 OO입니다</p>
    <ul>
      <li>취미: 산책과 사진</li>
      <li>목표: 멋진 웹사이트 만들기</li>
    </ul>
  </main>
  <footer>
    <a href="...">블로그 보러가기</a>
  </footer>
</body>
```

글·사진·목록·링크가 담긴 페이지 한 장 완성. 아직 CSS를 안 배워서 밋밋하지만, 뼈대는 다 갖췄다.

## 2. 어려웠던 것 — 심화 학습

### 시맨틱 태그, div로도 되는데 왜 굳이 쓸까?

수업에서 `<header>`, `<main>`, `<footer>`를 배우면서 "그냥 상자로 묶는 거면 다 똑같은 거 아닌가?"라는 의문이 들었다. 찾아보니 이유가 분명했다.

**택배 상자 비유:** 겉에 "주방용품"이라고 적힌 상자(시맨틱)가 라벨 없는 민무늬 상자(div)보다 나중에 찾기 쉽다. 이걸 알아보는 대상이 셋이나 된다.

1. **사람(개발자)** — 코드만 봐도 어디가 머리말이고 본문인지 바로 보인다.
2. **검색엔진** — 구글 봇이 `<main>` 안의 내용을 핵심 콘텐츠로 인식한다 → **SEO에 유리**
3. **화면낭독기(스크린 리더)** — 시각장애인 사용자가 "본문으로 바로 가기" 같은 탐색을 할 수 있다 → **웹 접근성**

즉 화면에 보이는 결과는 같아도, **기계와 사람 모두에게 구조를 설명해주는 것**이 시맨틱 태그의 존재 이유다.

### 블록 요소 vs 인라인 요소

태그마다 화면에서 자리를 차지하는 방식이 다르다는 것도 헷갈렸던 부분.

| 구분 | 특징 | 예시 |
|------|------|------|
| **블록 요소** | 벽돌처럼 한 줄을 통째로 차지, 위→아래로 쌓임 | `p`, `h1`, `ul`, `header` |
| **인라인 요소** | 줄을 바꾸지 않고 글자 사이에 끼어듦 | `a`, `strong`, `em` |

`<p>` 두 개를 나란히 써도 옆으로 붙지 않고 아래로 쌓이는 이유가 이것 때문이었다. 나중에 CSS 레이아웃 배울 때 이 개념이 계속 나온다고 하니 지금 확실히 잡아두자.
