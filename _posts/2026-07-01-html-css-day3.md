웹개발 수업 3일차. video·audio, Emmet, 개발자도구로 HTML을 마무리하고 미니 프로젝트로 첫 완성작을 만들었다. 그리고 드디어 CSS 시작 — 밋밋했던 페이지에 옷을 입히는 법을 배우기 시작했다.

## 1. 수업 내용 요약

### 📄 HTML 마무리 (STEP 29~34)

#### 1-1. 내 파일로 영상·음성 재생 (video, audio, source)

전에 배운 `<iframe>`은 유튜브·지도 같은 **남의 페이지**를 끼워 넣는 것. 내 파일을 직접 재생하려면 `<video>`와 `<audio>`를 쓴다.

```html
<video controls width="320" src="movie.mp4"></video>

<audio controls>
  <source src="song.mp3" type="audio/mpeg">  <!-- mp3 후보 -->
  <source src="song.ogg" type="audio/ogg">   <!-- ogg 후보 -->
  내 브라우저는 audio 미지원                  <!-- 둘 다 안 되면 보일 글 -->
</audio>
```

- **`controls`를 붙여야 재생 버튼이 뜬다.** 안 붙이면 화면에 아무것도 안 보여서 당황할 수 있음.
- `<source>`를 여러 개 적어두면 브라우저가 재생 가능한 형식을 알아서 골라 쓴다 (호환성 대비).

#### 1-2. Emmet — 짧게 쳐서 코드 빨리 만들기

VSCode에 기본 내장된 자동완성. 짧은 약어를 치고 `Tab`을 누르면 긴 HTML로 펼쳐진다.

| 약어 | 결과 | 기호 뜻 |
|------|------|---------|
| `!` | HTML 기본 뼈대 통째로 | 문서 틀 자동 생성 |
| `ul>li*3` | ul 안에 li 3개 | `>` 안에 넣기, `*N` N번 반복 |
| `.box` | `<div class="box">` | `.` = class |
| `#top` | `<div id="top">` | `#` = id |
| `h1{안녕}` | `<h1>안녕</h1>` | `{}` = 안에 들어갈 내용 |
| `a[href=#]` | `<a href="#">` | `[]` = 속성 |

`!` + Tab으로 생기는 뼈대에는 `<meta name="viewport" ...>` 줄이 포함되는데, **페이지 너비를 기기 화면에 맞추는 반응형의 출발점**이다. 없으면 폰에서 글자가 깨알같이 작게 나온다. (`lang="en"`으로 생성되니 `"ko"`로 바꾸는 것 잊지 말기!)

#### 1-3. 개발자도구 (F12) — 화면 뒤 들여다보기

크롬·엣지에 기본 내장된 실전 디버깅 도구. `F12` 또는 우클릭 → 검사로 연다.

- **화살표(검사) 아이콘** → 화면 요소에 마우스를 올리면 무슨 태그인지 짚어줌
- **Elements 탭** → 페이지 HTML 구조가 다 보임
- 글자·색 값을 **더블클릭해서 바꿔보기** → 화면에 바로 반영 (디자인 미리보기)
- **Console 탭**의 빨간 글씨 = 에러 메시지, 어디가 틀렸는지 힌트

여기서 바꾼 건 화면에서만 잠깐이고 내 파일은 안 바뀐다. 새로고침하면 원래대로라서 마음껏 실험해도 안전. **남의 잘 만든 사이트를 F12로 뜯어보는 게 가장 빠른 공부법**이라고 한다.

#### 1-4. 한 줄씩 질문하며 공부하기 — 물고기 잡는 법

STEP 01~32를 욱여넣은 어려운 더미 코드를 놓고, 모르는 줄을 **한 줄씩** Claude Code에 물어보며 빈칸을 채우는 연습을 했다. 질문 템플릿:

> 이 HTML 한 줄이 무슨 뜻인지 초등학생도 알게 설명해줘:
> `(모르는 코드 한 줄)`
> - 이걸 안 쓰면 화면이 어떻게 달라져?
> - 더 좋은 대안이나 같이 쓰면 좋은 게 있어?
> - 한 줄 요약으로 주석 달아줘

핵심은 답을 받는 게 아니라 **한 줄마다 질문을 만들어내는 습관**. 받은 답은 내 말로 줄여서 코드 옆 주석에 직접 적고, 값을 바꿔보고 F12로 확인한다.

#### 1-5. 미니 프로젝트 — 한 페이지짜리 내 소개 사이트 🎉

HTML 과정의 최종 관문. 배운 걸 전부 한 파일에 합쳐 작은 사이트를 완성했다.

- 시맨틱 구조: `header > nav` / `main > section` / `footer`
- `nav`의 `href="#about"` ↔ 각 section의 `id="about"`으로 **같은 페이지 안 점프**
- 표(공부 기록), 폼(연락처), 이미지, 목록까지 총동원

여기서 **id와 class의 실무 구분**이 확실해졌다:
- `id` = 페이지에 딱 하나 — 메뉴 점프(`#about`), `label`의 `for`↔`input`의 `id` 연결
- `class` = 여러 곳에 공유 — 같은 모양을 CSS 한 줄로 한 번에 꾸밀 때
- 한 태그에 둘 다 가능: `<section class="section" id="about">` (class=공유 스타일, id=고유 점프)

### 🎨 CSS 시작 (STEP 01~02)

#### 1-6. CSS란? 선택자 + 붙이는 법

CSS = 페이지에 옷 입히기(색·크기·위치). HTML이 뼈대라면 CSS는 색칠·꾸미기다.

```css
h1 { color: red; }
```

이 한 줄의 짜임:
- `h1` — **선택자** "누구를?"
- `color` — **속성** "무엇을?"
- `red` — **값** "어떻게?"
- 줄 끝 세미콜론 `;` 필수 — 빠지면 안 먹힌다!

CSS는 `styles/main.css`에 따로 적고, HTML `<head>`에서 `<link>`로 불러온다:

```html
<head>
  <link rel="stylesheet" href="styles/main.css" />
</head>
```

```
01/
├─ index.html      # 화면 (뼈대)
└─ styles/
   └─ main.css     # 꾸미기
```

**오늘 배운 선택자 3가지:**

```css
h1 { color: red; }         /* 요소 선택자 — 모든 h1 */
.special { color: orange; } /* 클래스 선택자 — class="special"만 */
li em { color: purple; }    /* 자손 선택자 — li 안의 em만 */
```

HTML에서 달아둔 `class`가 여기서 드디어 쓰인다. CSS 핵심 = **"누구를?(선택자) 어떻게?(속성과 값)"**

#### 1-7. 코드 순서 관례 (일반 → 구체)

CSS는 **넓은 것 먼저, 콕 집는 것 나중** 순서로 쌓는다.

```css
/* ① 기본 — 페이지 전체 */
body { background-color: lightyellow; }

/* ② 요소 선택자 */
h1 { color: navy; }
p { color: gray; }

/* ③ 클래스 선택자 */
.special { color: orange; }

/* ④ 자손 선택자 */
li em { color: purple; }
```

같은 속성이 충돌하면 **더 구체적인 선택자·더 아래에 있는 규칙이 이긴다.** 이 순서를 지키면 의도대로 동작한다. (외울 필요는 없다고 — 다음 시간에 배울 Stylelint가 자동 정렬해준다고 한다.)

## 2. 어려웠던 것 — 심화 학습

### CSS named color — 이름으로 부르는 색

`color: red;`처럼 색을 이름으로 쓸 수 있는데, 이런 **named color가 약 150개**나 정의되어 있다. 전체 목록은 MDN에서 확인:
👉 [MDN — named-color](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Values/named-color)

**동의어(같은 색, 다른 이름)도 있다:**

| 이름 A | 이름 B |
|--------|--------|
| `aqua` | `cyan` |
| `fuchsia` | `magenta` |
| `gray` | `grey` (영국식 철자) |

둘 다 완전히 같은 색이라 아무거나 써도 된다. `gray`/`grey`처럼 미국식·영국식 철자가 둘 다 등록된 경우도 있다 (`darkgray`/`darkgrey` 등도 마찬가지).

### 카키색(khaki)은 나라마다 다른 색이다?

named color를 보다가 `khaki`에서 흥미로운 사실을 발견했다. **같은 단어인데 나라마다 떠올리는 색이 다르다.**

| 지역 | '카키'가 가리키는 색 |
|------|---------------------|
| 영국식 (원조) | 황갈색, 녹갈색 — 군복에서 유래한 원래 의미 |
| 미국식 | 옅은 황갈색 (베이지에 가까움, 치노 팬츠 색) |
| 한국식 | 올리브색·국방색 — 사실 **오용**에 가까움 |

그럼 CSS의 `khaki`는 어느 쪽일까? 직접 확인해보니 **미국식에 가까운 옅은 황갈색(`#F0E68C`)**이다. 한국에서 생각하는 "카키색(국방색)"을 기대하고 `color: khaki;`를 쓰면 전혀 다른 연한 노란빛이 나온다!

```css
.usa-khaki { color: khaki; }          /* #F0E68C — 옅은 황갈색 */
.dark-khaki { color: darkkhaki; }     /* #BDB76B — 조금 더 어두운 황갈색 */
.korea-khaki { color: olive; }        /* #808000 — 한국인이 기대하는 '국방색'에 가까움 */
.army { color: darkolivegreen; }      /* #556B2F — 더 진한 군복 느낌 */
```

**교훈:** 색 이름은 문화권마다 가리키는 색이 다를 수 있으니, 정확한 색이 필요하면 이름 대신 `#F0E68C` 같은 코드값(hex)을 쓰는 게 안전하다. 이름은 어디까지나 "대충 이 계열" 정도로.

> 여담: 수업 예제에 나온 `rebeccapurple`(#663399)은 CSS 표준 제정자 Eric Meyer의 딸 Rebecca를 기리며 추가된 색이라고 한다. named color에도 이야기가 담겨 있다.
