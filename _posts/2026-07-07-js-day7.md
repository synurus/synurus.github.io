웹개발 수업 7일차. 이벤트로 사용자 행동에 반응하고, localStorage로 데이터를 기억하고, 반복문·배열·객체·클래스로 데이터를 다루는 법까지. 마지막엔 fetch로 서버와 데이터를 주고받는 CRUD까지 나갔다. JS가 "장난감"에서 "도구"로 바뀌기 시작한 날. 심화에서는 백틱과 작은따옴표의 차이, 디버깅 단축키를 정리했다.

## 1. 수업 내용 요약

### 1-1. 이벤트 (Events) — 사용자 행동에 반응하기

이벤트 = 클릭·키 누르기 같은 사용자 행동. "이 행동이 일어나면 이걸 해줘"라고 미리 약속해두면 페이지가 반응한다.

```js
let n = 0;                          // 값이 변하니 let
btn.addEventListener("click", () => {
  n++;                              // n = n + 1 과 같음
  out.textContent = `${n}번 눌렀어요`;
});
```

`click` 말고도 `mouseover`(마우스 올림), `keydown`(키 누름) 등 종류가 많다.

### 1-2. 이미지 바꾸기 — 클릭하면 그림 토글

지금 그림이 무엇인지 확인하고(`getAttribute`) 반대 그림으로 교체(`setAttribute`)하는 패턴.

```js
myImage.onclick = () => {
  const mySrc = myImage.getAttribute("src");   // 지금 주소 읽기
  if (mySrc === IMG_A) {
    myImage.setAttribute("src", IMG_B);        // A였으면 → B로
  } else {
    myImage.setAttribute("src", IMG_A);        // 아니면 → A로
  }
};
```

**💡 onclick vs addEventListener — 같은 클릭 처리, 두 방법:**

- `onclick` — 새로 쓰면 앞 함수가 **덮어써져** 마지막 1개만 남음
- `addEventListener` — 등록할 때마다 **쌓여서** 등록 순서대로 전부 실행 → 실무 권장

```js
btn.onclick = 저장하기;
btn.onclick = 알림띄우기;              // 저장하기 사라짐 → 알림만 실행

btn.addEventListener("click", 저장하기);
btn.addEventListener("click", 알림띄우기); // 클릭 1번 → 저장 + 알림 둘 다
```

### 1-3. 환영 메시지 저장 — localStorage

브라우저의 작은 메모장. 저장해두면 **새로고침하거나 다음에 다시 와도 기억**한다.

```js
localStorage.setItem("name", myName);   // 저장
localStorage.getItem("name");           // 꺼내기 (없으면 null)
localStorage.removeItem("name");        // 지우기
```

페이지를 처음 열 때 `getItem`으로 저장된 이름이 있는지 확인하고, 있으면(`if (saved)`) 바로 인사하는 흐름.

### 1-4. 반복문 (Loops) — 똑같은 일을 여러 번

```js
// for — 세 가지(초기화; 조건; 증감)가 괄호 안 한 줄에
for (let i = 1; i <= count; i++) {
  const li = document.createElement("li");
  li.textContent = `${i}번째 🍎`;
  out.appendChild(li);
}

// while — 조건이 참인 동안 반복 (초기화는 위, 증감은 본문 안)
let i = Number(n.value);
while (i > 0) {
  /* ... */
  i--;   // ⚠️ 안 줄이면 무한반복!
}
```

새로 만난 화면 조작 도구: `createElement("li")` = 요소 만들기, `appendChild(li)` = 붙이기, `innerHTML = ""` = 비우기.

### 1-5. 문자열 메서드 — 글자 다루는 도구

글자에는 도구가 딸려 있다. 점(`.`)을 찍고 도구 이름을 부르면 된다.

```js
text.length              // 글자 수 — 괄호 없는 속성
text.toUpperCase()       // 모두 대문자로 — 괄호 붙는 메서드
text.replace("e", "E")   // 일부를 다른 글자로
```

어제 배운 구분법 그대로: **괄호 `()` 있으면 메서드, 없으면 속성.**

### 1-6. 배열 메서드 — 여러 값을 한 줄에

배열은 사물함처럼 값 여러 개를 칸칸이 담는 묶음.

```js
const fruits = ["사과", "바나나"];
fruits.push(fruit.value);        // 배열 끝에 새 값 추가 (메서드)
fruits.length                    // 담긴 개수 (속성)
fruits.join(", ")                // "사과, 바나나" — 이어붙여 한 문장으로
```

**두 가지 중요한 패턴:**

- 배열인데 왜 `const`? — `push`로 **내용**이 바뀌어도 변수 자체를 재할당하는 게 아니라서 `const`로 선언한다.
- **render 패턴** — 화면을 다시 그리는 함수를 하나 만들어두고, 데이터가 바뀔 때마다 호출해 화면을 갱신한다. 데이터와 화면을 분리하는 첫걸음.

### 1-7. 객체 (Objects) — 한 덩어리로 묶기

서로 관련된 값들을 이름표 붙여 한 덩어리로 묶은 것.

```js
const person = {
  name: "콩이",    // 이름표(속성) name → 값 "콩이"
  age: 10
};

person.name      // 점(.)으로 값 읽기 → "콩이"
person.age++     // 점 표기로 값 바꾸기
```

이름표 하나하나를 **속성(property)**이라 부른다. `document.querySelector(...)`가 돌려주던 것도 사실 객체였다.

### 1-8. 클래스 (Class) — 객체 찍어내는 틀

객체를 하나하나 손으로 만들면 번거로우니, **같은 모양 객체를 여러 개 찍어내는 틀(붕어빵 틀)**이 클래스. `new`로 찍으면 객체(인스턴스)가 나온다.

```js
class Dog {
  constructor(name) {      // new로 만들 때 자동 호출
    this.name = name;      // this = 지금 만들어지는 객체 자신
  }
  bark() {                 // 메서드 — 모든 Dog 객체가 함께 쓰는 동작
    return `${this.name}: 멍멍!`;
  }
}

const poppy = new Dog("뽀삐");
const choco = new Dog("초코");
poppy.bark();   // "뽀삐: 멍멍!"
```

재밌는 연결: `Document`(대문자)는 클래스, 우리가 계속 쓰던 `document`(소문자)는 그 틀로 찍은 객체였다.

### 1-9. 디버깅 (Debugging) — 에러 잡고 값 확인하기

코드가 안 될 때 **추측은 금물**. F12 → Console 탭에서 값을 직접 확인한다.

```js
console.log("price:", price);   // JS의 프린트 — 이름표 붙이면 안 헷갈림
console.log(typeof price);      // 값의 종류 확인
debugger;                       // 이 줄에서 실행이 멈춤 (F12 열려 있을 때만)
```

왜 중요한가 — 코드는 눈에 안 보이게 돌아가서, 찍어봐야 "내 생각 vs 실제 값"이 맞는지 알 수 있다. 버그 대부분이 "값이 내 예상과 다름"이라 출력 한 줄이 원인을 바로 드러낸다.

**자주 만나는 에러 3종 (한글 번역기):**

| 에러 메시지 | 뜻 |
|------------|-----|
| `Cannot read properties of null` | querySelector가 못 찾음 → id 오타·script 위치 확인 |
| `X is not defined` | 변수·함수 이름 오타거나 선언 안 함 |
| `X is not a function` | 오타거나 그 값에 없는 메서드 호출 |

6일차에 배운 "JS를 `</body>` 앞에 두는 이유"가 첫 번째 에러와 바로 연결된다.

### 1-10. 서버 연동 (CRUD) — fetch로 데이터 주고받기

지금까지는 localStorage(내 브라우저에만 저장)였다면, 진짜 서버와는 `fetch`로 데이터를 주고받는다. **json-server**로 가짜 서버를 띄워 연습했다.

**CRUD ↔ HTTP method 짝:**

| 동작 | 뜻 | method |
|------|-----|--------|
| **C**reate | 생성 | `POST` |
| **R**ead | 조회 | `GET` |
| **U**pdate | 수정 | `PUT` |
| **D**elete | 삭제 | `DELETE` |

```js
// READ — 목록 조회
document.querySelector("#btn-read").addEventListener("click", async () => {
  const res = await fetch(API);
  const users = await res.json();   // 응답을 객체로 변환
  show("READ 결과", users);
});

// CREATE — 새 사용자 추가
const res = await fetch(API, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "John" })   // 보낼 땐 객체 → 글자
});
```

**핵심 개념:**

- `JSON.stringify(객체)` — 보낼 때 객체를 글자로 / `res.json()` — 받은 응답을 다시 객체로
- **fetch는 비동기** — 응답이 언제 올지 몰라서 브라우저를 멈춰놓고 기다리지 않는다 (그래야 기다리는 동안 클릭·스크롤이 계속 됨)
- **`await`("기다리다")** — 그 결과가 올 때까지 이 줄만 대기
- **`async`("비동기")** — 함수 앞에 붙여 "이 안에서 await 씀"을 표시하는 팻말. **await은 async 함수 안에서만** 쓸 수 있다 (짝꿍)
- 실행 주의: json-server는 전역 설치 말고 폴더 안에서 `npm install` 후 **`npm start`**로 실행 (전역 최신 버전은 id를 랜덤 문자열로 만들어 예제와 안 맞음)
- 실제 서비스에서는 이 자리에 진짜 백엔드(Node.js·Express 등)가 들어간다

## 2. 어려웠던 것 — 심화 학습

### 2-1. 백틱(`)과 작은따옴표(')의 차이

템플릿 리터럴을 쓰다가 따옴표 종류가 헷갈려서 정리했다.

**`'` 작은따옴표 — 일반 문자열**

```js
const name = '철수';
const msg = '안녕하세요, ' + name + '님';   // 변수를 넣으려면 + 로 연결
```

- 그냥 텍스트를 담는 용도. `"` 큰따옴표와 기능이 완전히 같다.
- 여러 줄을 쓸 수 없고, 변수를 넣으려면 `+`로 이어 붙여야 한다.

**`` ` `` 백틱 — 템플릿 리터럴 (ES6)**

```js
const name = '철수';
const msg = `안녕하세요, ${name}님`;   // ${} 안에 변수/식을 바로 넣음
```

백틱만 할 수 있는 것 두 가지:

**① `${}` 문자열 보간** — 변수뿐 아니라 **계산식**도 직접 삽입할 수 있다.

```js
`합계: ${price * count}원`
```

**② 줄바꿈 그대로 유지 — 여러 줄 문자열**

```js
const html = `
  <div class="card">
    <p>${name}</p>
  </div>
`;
```

특히 JS로 HTML 조각을 만들 때 아주 유용하다. (오늘 배운 render 패턴에서 화면을 그릴 때 바로 쓰이는 기술!)

**한눈 정리:**

| | `'문자열'` | `` `문자열` `` |
|---|-----------|---------------|
| 변수 삽입 | `+` 연결 필요 | `${변수}` 바로 사용 |
| 여러 줄 | 불가 (`\n` 필요) | 그대로 가능 |

**참고:**
- 키보드 위치: `'`는 Enter 옆, `` ` ``는 숫자 1 왼쪽 (한/영 전환키 위)
- 스타일 관례: 변수 없는 단순 문자열엔 `'`, 변수가 들어가면 `` ` ``를 쓰는 게 일반적

### 2-2. 디버깅 단축키 — debugger에서 멈춘 다음엔?

수업에서 `debugger`로 실행을 멈추는 것까지 배웠는데, **멈춘 다음 어떻게 움직이는지**가 진짜 디버깅이다. 개발자도구(Sources 패널)의 단축키:

| 단축키 | 이름 | 동작 |
|--------|------|------|
| `F5` | Resume | 다음 중단점으로 계속 진행 |
| `F10` | Step Over | 한 줄씩 실행 (함수 내부로 안 들어감) |
| `F11` | Step Into | 함수 내부로 한 단계 진입 |
| `Shift + F11` | Step Out | 현재 함수를 빠져나감 |
| `F8` | — | 디버깅 모드 종료 (또는 다음 중단점까지 실행) |

**Step Over vs Step Into가 핵심 구분:** 이 줄에 함수 호출이 있을 때 —
- 그 함수는 믿고 결과만 볼래 → `F10` (Step Over)
- 그 함수 **안**이 의심스러워 → `F11` (Step Into)

`console.log`를 여러 개 찍는 것보다, 중단점 하나 걸고 F10으로 한 줄씩 따라가며 변수값을 보는 게 훨씬 강력하다.
