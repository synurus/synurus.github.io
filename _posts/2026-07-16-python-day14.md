## 1. 수업 내용 요약 ① — Anaconda 환경 설정

### 1-1. Anaconda란? — 프로젝트마다 독립된 "방"

집에 주방·화장실·거실이 따로 있듯, 코딩 프로젝트도 **프로젝트마다 필요한 라이브러리와 버전이 다르다.** Anaconda는 프로젝트별로 독립된 방(**가상환경**)을 만들어 서로 간섭 없이 분리해주는 관리자다.

> Node 세계와 번역하면: 프로젝트 폴더마다 `node_modules`가 따로 있던 것과 같은 철학. 다만 Python은 "폴더"가 아니라 "환경"을 만들어 갈아탄다.

설치는 아카이브에서 exe 다운로드 → **"Add Anaconda3 to my PATH" 체크**가 포인트. 확인은 `conda --version`.

### 1-2. 가상환경 생성 & 관리 — 오늘의 핵심 명령어

```bash
conda create -n cv_env python=3.10   # 환경 만들기 (이름·파이썬 버전 지정)
conda activate cv_env                # 방에 들어가기
conda deactivate                     # 방에서 나오기
conda env list                       # 방 목록 보기
conda env remove -n cv_env           # 방 없애기
```

프로젝트마다 파이썬 버전까지 다르게 쓸 수 있다는 게 인상적 (`cv_env`는 3.10, `nlp_env`는 3.9 하는 식).

### 1-3. 패키지 설치와 환경 공유

```bash
pip install pyfiglet                  # 설치 (npm install의 파이썬판)
pip install package==1.2.3            # 특정 버전
pip install -r requirements.txt       # 목록 파일로 한 번에
pip list                              # 설치 목록

pip freeze > requirements.txt         # 내 환경의 의존성 내보내기
conda env export > environment.yml    # conda 방식 내보내기
conda env create -f environment.yml   # 남의 환경 그대로 복원
```

> `requirements.txt` = `package.json`의 dependencies와 같은 역할 — "이 프로젝트 돌리려면 이거 깔아" 목록. 팀원과 같은 환경을 맞추는 실무 기본.

## 2. 수업 내용 요약 ② — Python 문법 (JS와 번역하며)

### 2-1. 변수 타입 9종 — JS보다 그릇이 다양하다

```python
x = 10                              # int (정수)
y = 3.14                            # float (실수)
name = "Python"                     # str
is_fun = True                       # bool — 대문자 T/F 주의!
colors = ["red", "green", "blue"]   # list  — JS의 배열
coords = (10, 20)                   # tuple — 수정 불가 리스트 (JS에 없음)
person = {"name": "Tom", "age": 30} # dict  — JS의 객체
nums = {1, 2, 3}                    # set   — 중복 불가 (JS에도 Set 있음)
nothing = None                      # NoneType — JS의 null
```

- 타입 확인은 `type(x)`, JS의 `typeof`와 같은 역할
- **JS와 다른 점 3연타:** `let/const` 없이 그냥 대입 · `True/False` 대문자 · `None`
- **네이밍이 문법 수준의 관례** — 변수·함수는 `snake_case`(PEP 8 공식 권장), 클래스는 `PascalCase`. JS에서 쓰던 `camelCase`는 파이썬에선 비권장!
- 예약어(`if`, `class`, `lambda` 등)는 변수명 금지 — JS와 같은 규칙

### 2-2. 연산 — 새 얼굴 셋

```python
print(a / b)    # 2.5  나눗셈 (항상 실수)
print(a // b)   # 2    몫 (JS엔 없음 — Math.floor(a/b) 하던 것)
print(a % b)    # 1    나머지
print(a ** b)   # 25   거듭제곱 (JS도 ** 있지만 여기가 원조 느낌)
print("hello" * 2)   # hellohello — 문자열 곱하기 반복! (JS엔 없음)
score += 10     # 복합 대입은 JS와 동일
```

### 2-3. 리스트 — 배열의 파이썬판, 그런데 더 편하다

```python
colors[0]       # 첫 번째
colors[-1]      # 마지막! — JS의 arr[arr.length-1]이 한 글자로
colors[0:2]     # 슬라이싱 — JS의 slice(0, 2)가 문법으로
colors.append("pink")     # push
colors.insert(0, "white") # 특정 위치 삽입
colors.remove("white")    # 값으로 제거
numbers.sort()            # 정렬 (내림차순은 reverse=True)
print(2 in numbers)       # True — includes()가 in 키워드로
```

**음수 인덱스(`[-1]`)와 슬라이싱**은 JS로 돌아가면 그리워질 문법.

### 2-4. f-string — 템플릿 리터럴의 파이썬판

```python
print(f"저는 {name}입니다. 나이는 {age}살입니다.")
print(f"{a} + {b} = {a + b}")            # 계산식도 OK
print(f"소수점 2자리: {3.14159:.2f}")     # 3.14 — 포맷 지정까지!
```

JS: `` `안녕, ${name}` `` → Python: `f"안녕, {name}"`. 백틱 대신 **f 접두사**, `${}` 대신 `{}`. `:.2f` 같은 포맷 지정은 파이썬이 한 수 위.

### 2-5. 조건문과 반복문 — 중괄호가 사라졌다

```python
if temperature >= 30:
    print("덥다! 반팔 입기")
elif temperature >= 20:          # else if → elif
    print("딱 좋아! 가볍게 입기")
else:
    print("추워!")
```

**가장 큰 충격: 중괄호 `{}`가 없다.** 블록은 **들여쓰기 자체가 문법**이다. JS에선 들여쓰기가 "예의"였다면 파이썬에선 "법률". 콜론(`:`) 뒤 들여쓰기 4칸이 국룰.

```python
for item in mixed:                          # for...of 느낌
    print(item)

for index, item in enumerate(mixed):        # 인덱스+값 동시에
    print(f"index: {index}, item: {item}")

while number > 0:                           # while은 JS와 동일
    number -= 1

while True:                                 # 무한 반복 + break 탈출
    ...
    break
```

### 2-6. zip() — 두 리스트를 지퍼처럼 짝짓기 (새 개념!)

```python
names = ["뽀삐", "초코", "쿠키"]
scores = [95, 88, 72]

for name, score in zip(names, scores):      # 한 바퀴에 한 쌍씩
    print(f"{name}: {score}점")

pairs = list(zip(names, scores))    # [('뽀삐', 95), ('초코', 88), ...]
person = dict(zip(keys, values))    # 키 리스트 + 값 리스트 → 딕셔너리 한 줄!
```

- 담는 그릇만 고르면 된다: `list()` = 줄 세워 번호로 꺼냄, `dict()` = 이름표 붙여 이름으로 꺼냄
- 길이가 다르면 짧은 쪽 기준으로 끝남

### 2-7. 외부 라이브러리 첫 사용 — pyfiglet + termcolor

```python
import pyfiglet
from termcolor import colored

py_text = pyfiglet.figlet_format("Python")            # 아스키 아트로
color_text = colored(py_text, "yellow", "on_blue", ["bold"])  # 색 입히기
print(color_text)
```

- `import 라이브러리` 통째로 / `from 라이브러리 import 함수` 골라서 — JS의 `import { colored } from 'termcolor'` 감각 그대로
- 터미널에 대문짝만한 컬러 글씨가 뜨는 게 소소하게 재밌다

### 2-8. 함수 — def, return, 그리고 문서화 습관

```python
def good_sentence(sentence: str) -> None:
    """
    입력된 문자열을 pyfiglet 형식으로 출력합니다.
    매개변수: sentence (str) / 반환: None
    """
    print(pyfiglet.figlet_format(sentence))
```

- `def 이름(재료):` — JS의 `const fn = (재료) => {}` 대응
- **타입 힌트** — 매개변수 `: str`, 반환 `-> None`. 강제는 아니지만 안내판 역할
- **독스트링(`"""..."""`)** — 함수 첫 줄의 설명서. 주석 습관이 문법으로 자리 잡은 느낌
- `return`은 JS와 동일 — print는 "보여주기", return은 "건네주기"

### 2-9. 예외 처리 — try/except

```python
try:
    meters = float(user_input)       # 변환 시도
    print(f"{meters}m는 {meters * 3.28084:.2f}ft입니다.")
except ValueError:                   # 실패하면 여기로
    print("숫자를 입력해주세요.")
```

JS의 `try/catch`가 `try/except`로. 다른 점은 **오류 종류별로 잡는 게 기본**이라는 것 — `ValueError`(값 형식 오류), `ZeroDivisionError`(0 나누기), `TypeError`(타입 불일치). `while True` + `try/except` + `break` 조합으로 "올바른 입력 받을 때까지 재시도" 패턴도 배웠다.

### 2-10. 파일 입출력 — with는 자동으로 닫아주는 센서 수전

```python
with open("memo.txt", "w", encoding="utf-8") as f:   # w = 새로 쓰기(덮어씀)
    f.write("안녕, 파이썬!\n")
# 블록을 나오면 파일이 알아서 닫힘

with open("memo.txt", "r", encoding="utf-8") as f:   # r = 읽기
    text = f.read()

with open("memo.txt", "a", encoding="utf-8") as f:   # a = 이어 쓰기
    f.write("새로운 한 줄 추가\n")
```

- 모드 세 개만 기억: **w**(덮어쓰기) / **r**(읽기) / **a**(append, 덧붙이기)
- `encoding="utf-8"` — 한글 깨짐 방지 (HTML의 `<meta charset>`과 같은 이유!)
- 브라우저 JS는 보안상 파일을 못 만졌는데(12일차 배운 것), 파이썬은 로컬 파일을 자유롭게 읽고 쓴다 — 서버 쪽 언어라는 게 실감나는 지점

### 2-11. 클래스 — 뽀삐와 초코의 재회 🐕

```python
class Dog:                            # 붕어빵 틀 (설계도)
    def __init__(self, name, age):    # 생성자 — 만들 때 자동 실행
        self.name = name              # 자기 이름 저장
        self.age = age

    def bark(self):                   # 메서드 — 할 수 있는 일
        print(f"{self.name}: 멍멍!")

dog1 = Dog("뽀삐", 3)                 # new 없이 바로 호출!
dog2 = Dog("초코", 5)
dog1.bark()                           # 뽀삐: 멍멍!
```

7일차 JS 클래스의 Dog 예제(뽀삐·초코)가 그대로 파이썬으로 돌아왔다. 번역표:

| JS | Python |
|-----|--------|
| `constructor(name)` | `__init__(self, name)` |
| `this.name` | `self.name` |
| `new Dog("뽀삐")` | `Dog("뽀삐")` — new 없음 |

수업에서 좋았던 건 **"함수만으로 만들면 어떻게 되나"** 비교 — 강아지마다 `dog1_name`, `dog1_age`... 변수가 흩어지고, 100마리면 변수 200개 폭발. 클래스는 **정보(속성)와 행동(메서드)을 한 통에** 묶어서 이걸 해결한다. "왜 클래스인가"에 대한 답이 명확해졌다.

### 2-12. 람다 — 화살표 함수의 사촌

```python
add = lambda a, b: a + b                              # 양식: lambda 재료: 결과
decorate_text = lambda text: pyfiglet.figlet_format(text)
```

JS의 `const add = (a, b) => a + b`와 정확히 같은 자리. 일반 함수는 **긴 편지**, 람다는 **짧은 쪽지** — 한 줄짜리 일회용 함수에 어울린다.

## 3. 심화 — Teachable Machine으로 첫 머신러닝 🤖

[Teachable Machine](https://teachablemachine.withgoogle.com/)에서 **웹캠 영상으로 이미지 프로젝트 학습**을 진행해봤다. 머신러닝 프로젝트는 처음이라 신기했고, **생각보다 학습이 훨씬 잘 돼서 놀랐다.**

### 뭘 해본 건가

Teachable Machine = 구글이 만든 **코드 없는 머신러닝 도구.** 브라우저에서:

1. 클래스(분류할 그룹)를 만들고 — 예: "얼굴", "손", "배경"
2. 웹캠으로 각 클래스의 예시 이미지를 수십 장씩 찍고
3. **Train 버튼 한 번** — 몇 초~몇 분 만에 학습 완료
4. 웹캠을 다시 비추면 실시간으로 "지금 화면은 ○○ 클래스, 확신도 97%"를 보여준다

### 왜 이렇게 잘 배우지? — 찾아본 뒷이야기

몇십 장만 보여줬는데 분류가 잘 되는 게 의아해서 찾아봤다. 비밀은 **전이 학습(Transfer Learning)** — 밑바닥부터 배우는 게 아니라, 구글이 이미 수백만 장의 이미지로 훈련해둔 모델(MobileNet)을 가져와 **마지막 판단 부분만 내 데이터로 다시 배우는** 방식이라고 한다. 비유하면, 그림을 볼 줄 아는 화가에게 "이 두 그림만 구분해줘"라고 부탁하는 것 — 이미 눈이 트여 있으니 예시 몇 장이면 충분한 것.

그리고 학습이 **브라우저 안에서**(TensorFlow.js) 돌아간다 — 내 웹캠 영상이 서버로 전송되지 않는 구조라는 것도 알게 됐다.

### 웹개발과 연결하면

학습이 끝난 모델은 **Export → TensorFlow.js**로 내보낼 수 있다 — 즉, **내가 만든 웹사이트에 embed할 수 있다.** JS로 모델을 불러와 실시간 웹캠 분류를 붙이는 것. 픽셀 사이트나 차트 앱에 "카메라에 손이 보이면 반응하는" 기능을 넣는 것도 이론상 가능하다는 뜻이라, 언젠가 미니 프로젝트 확장 아이디어로 킵.

> 오늘 Python을 시작한 것과 이 체험이 이어진다 — 머신러닝의 표준 언어가 바로 Python이다. Teachable Machine이 버튼으로 해준 그 학습을 코드로 직접 하는 게 아마 앞으로의 진도일 것.
