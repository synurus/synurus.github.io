10일차에 몸살로 조퇴하면서 놓쳤던 React 32~39번을 따로 따라잡았다. 하필 놓친 구간이 **state의 실전(폼 입력)과 useEffect 전부**라서, 수업 페이지를 보며 하나씩 직접 쳐보고 정리한다.

## 1. 폼 입력 — input 값을 state로 관리 (controlled)

입력창에 친 글자를 React가 state로 붙잡아 실시간으로 쓰는 패턴. 검색·로그인·댓글의 기본.

```jsx
const NameForm = () => {
  const [name, setName] = useState('')   // 입력값 보관 (처음 빈 문자열)
  return (
    <>
      <input value={name} onChange={(e) => setName(e.target.value)} />
      <p>안녕, {name}!</p>                {/* 입력 즉시 화면 반영 */}
    </>
  )
}
```

- **`value`(칸에 보일 값) + `onChange`(칠 때마다 갱신)는 한 쌍** — 둘 다 있어야 입력이 된다
- `e.target.value` = 지금 입력칸의 글자 (JS에서 쓰던 `.value`가 이벤트 객체를 통해 오는 것)
- 입력값이 state라서 검증·전송·초기화를 코드로 자유롭게 처리 — 이걸 **controlled(제어) 컴포넌트**라 부른다

## 2. ProductItem — props와 state의 역할 구분 ★

한 부품이 **props와 state를 둘 다** 가질 수 있다. 둘의 차이가 이 구간의 핵심.

```jsx
const ProductItem = ({ name }) => {            // name = props (부모가 줌)
  const [count, setCount] = useState(0)        // count = state (자기가 기억)
  return (
    <div>
      <h3>{name}</h3>
      <p>{count}개 담음</p>
      <button onClick={() => setCount(c => c + 1)}>🛒 담기</button>
    </div>
  )
}

<ProductItem name="러닝화" />   // name만 넘김 — count는 부품 내부 소관
```

| | props | state |
|---|-------|-------|
| 누가 정하나 | 부모가 줌 | 자기가 기억 |
| 바꿀 수 있나 | ❌ 읽기 전용 | ✅ set 함수로 |
| 성격 | 외부 입력 | 내부 기억 |

## 3. useEffect 3형제 — 두 번째 칸이 전부다

**useEffect = 화면이 그려진 "뒤"에 실행할 일** (타이머·fetch 같은 화면 밖 작업). 두 번째 칸(의존성 배열)에 따라 실행 시점이 갈린다.

| 두 번째 칸 | 언제 실행 | 용도 |
|-----------|----------|------|
| `[]` | 처음 딱 1번 | 시작 설정·데이터 가져오기 |
| `[count]` | count 바뀔 때마다 (+처음 1번) | 값 추적·자동 저장·검색어 반응 |
| (생략) | 매 렌더마다 | 무한 반복 위험 — 실무 거의 안 씀 |

```jsx
useEffect(() => {
  console.log('화면 뜰 때 딱 1번')
}, [])

useEffect(() => {
  console.log('count 바뀜:', count)
}, [count])
```

### 심화 예제: 실시간 시계 — cleanup의 등장

```jsx
const Clock = () => {
  const [time, setTime] = useState(new Date().toLocaleTimeString())

  useEffect(() => {
    const timer = setInterval(() => {
      setTime(new Date().toLocaleTimeString())   // 1초마다 갱신 → 리렌더
    }, 1000)
    return () => clearInterval(timer)            // ★ 사라질 때 타이머 정리 (cleanup)
  }, [])                                          // 처음 1번만 등록

  return <p>⏰ {time}</p>
}
```

- `setInterval(콜백, 1000)` = 1초마다 반복 (브라우저 내장)
- **`return () => ...` = cleanup 함수** — 컴포넌트가 사라질 때 실행된다. 타이머를 안 끄면 화면에서 사라진 뒤에도 계속 돌아 메모리 낭비!
- "등록한 건 정리한다" — 미니 프로젝트에서도 fetch 경합 방지에 이 패턴을 썼다 (보충 후 프로젝트에서 바로 써먹음)

## 4. 데이터 가져오기 — fetch + useEffect

"화면 뜨면 → 데이터 요청 → 도착하면 화면 채움" 흐름. 7일차 JS에서 배운 fetch가 React 안으로 들어왔다.

```jsx
const Users = () => {
  const [users, setUsers] = useState([])        // 도착 전엔 빈 배열

  useEffect(() => {
    const fetchData = async () => {
      try {
        const res = await fetch('https://jsonplaceholder.typicode.com/users')
        const data = await res.json()
        setUsers(data)                           // 도착하면 채움 → 화면 갱신
      } catch (error) {
        console.error('데이터 로딩 실패:', error)
      }
    }
    fetchData()
  }, [])                                          // 첫 렌더 1번만 요청

  return (
    <ul>
      {users.map((u) => <li key={u.id}>{u.name}</li>)}
    </ul>
  )
}
```

핵심 흐름: **처음엔 빈 배열로 그림 → 도착하면 `setUsers` → React가 다시 그림.** 비동기라 "멈춰서 기다리지 않고, 도착하면 그때 처리"라는 감각이 JS CRUD 때와 똑같다.

## 5. 날씨 API — 같은 패턴, 주소만 교체

fetch 패턴이 손에 익었는지 확인하는 예제. 구조는 4번과 완전히 같고 주소와 값 꺼내는 경로만 다르다.

```jsx
const [temp, setTemp] = useState(null)           // 도착 전엔 null
const [isLoading, setIsLoading] = useState(true) // 로딩 상태도 state로!

// https://api.open-meteo.com/v1/forecast?latitude=37.5&longitude=127&current_weather=true
//   ? = 옵션 시작, & = 옵션 이어붙이기, latitude/longitude = 서울 좌표
setTemp(data.current_weather.temperature)        // 점(.) 타고 상자 속 상자 파고들기
```

```jsx
return <p>🌡️ 서울 기온: {isLoading ? '불러오는 중…' : (temp ? temp + '°C' : '불러올 수 없음')}</p>
```

- **로딩 상태도 state** — `isLoading`으로 "불러오는 중…"을 먼저 보여주는 게 실무 기본
- URL 뒤 `?key=value&key=value`가 **쿼리스트링** — 요청에 조건을 실어 보내는 방법
- 위도·경도만 바꾸면 다른 도시 (부산 35.1 / 129.0)

## 정리 — 놓친 구간이 사실 제일 중요했다

32~39는 결국 한 문장이었다: **"기억은 useState, 바깥일은 useEffect, 데이터는 fetch로 가져와 state에 담는다."** 이 세 개가 곧바로 미니 프로젝트의 뼈대가 됐으니, 아파도 따라잡길 잘했다. 💪

---

**보충의 한 줄 정리:** useEffect의 두 번째 칸 — `[]`는 한 번, `[값]`은 그 값 바뀔 때, 생략은 매번. 그리고 등록한 것은 cleanup으로 정리한다!
