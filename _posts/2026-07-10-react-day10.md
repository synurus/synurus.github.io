## 1. 수업 내용 요약

### 1-1. Alert — Props 객체 매핑

type이 여러 개일 때 삼항 연산자를 길게 중첩하지 않고, **객체에서 키로 골라 쓰는** 패턴.

```jsx
const Alert = ({ type, text }) => {
  const map = {                                    // 종류별 설정 모음(객체)
    success: { icon: '✅', color: 'green' },
    error:   { icon: '❌', color: 'crimson' },
    warning: { icon: '⚠️', color: 'orange' },
  }
  const cfg = map[type]                            // type을 키로 설정 꺼냄
  return <p style={{ color: cfg.color }}>{cfg.icon} {text}</p>
}
```

흐름 4단계: ① `type`(종류)를 키로 삼아 → ② `map[type]`로 설정 꺼냄 → ③ 꺼낸 색·아이콘으로 → ④ 화면에 그림. 삼항 3중첩보다 **읽기 쉽고 종류를 늘리기도 쉽다** — 알림·뱃지처럼 종류 많은 UI에 딱.

### 1-2. Rating & Tag — 배열이 화면이 되는 법 (map + key)

**Rating — 숫자 하나로 별 5칸 그리기:**

```jsx
const Rating = ({ score }) => (
  <div>
    {[...Array(5)].map((_, i) => (              // 빈칸 5개 배열 만들어 반복
      <span key={i}>{i < score ? '⭐' : '☆'}</span>   // i가 score보다 작으면 ⭐
    ))}
  </div>
)
// <Rating score={3} />  →  ⭐⭐⭐☆☆
```

`[...Array(n)].map` = n번 반복 화면 만들기 공식. `_`는 "칸의 값은 안 쓸게"라는 관례 표기.

**Tag — 배열 props를 목록으로 (실무 데이터의 기본형):**

```jsx
const Tag = ({ tags }) => (
  <div>
    {tags.map((tag) => (
      <span key={tag}>{'#' + tag}</span>
    ))}
  </div>
)
// <Tag tags={['react', 'props', 'map']} />  →  #react #props #map
```

- **배열 = 재료 목록, `map` = 자동 찍어내는 틀** — 재료 N개 → 화면 요소 N개
- 각 항목엔 **`key` 필수** (항목 이름표, 없으면 경고). Rating처럼 고정 5칸은 index도 괜찮지만, 실제 동적 목록은 고유 id를 써야 한다고.
- JS에서 배운 `map`이 React에선 "배열 → 화면"의 핵심 도구가 됐다.

### 1-3. MUI — 잘 만든 컴포넌트 갖다 쓰기

버튼·입력창을 직접 안 만들고 **완성품을 import**해서 쓰는 UI 라이브러리 (구글 머티리얼 디자인).

```bash
npm install @mui/material @emotion/react @emotion/styled
```

```jsx
import { Button } from '@mui/material'          // 기성 부품 가져옴

<Button variant="contained" onClick={() => alert('안녕!')}>
  제출
</Button>                                        // variant=모양, onClick=동작 → 다 props
```

- **라이브러리** = 자주 쓰는 부품을 미리 만들어 모아둔 꾸러미. 설치해서 import로 가져다 씀
- `Button`, `TextField`, `Select`, `Card`, `Dialog`, `AppBar`, `Snackbar` 등 전부 `import { 이름 } from '@mui/material'` 한 줄
- 직접 CSS 안 짜도 디자인 일관성 확보. 비슷한 것: Ant Design, Chakra UI, shadcn/ui

### 1-4. 컴포넌트 위치 — 적은 자리가 곧 화면 자리

"부품을 원하는 위치에 끼운다" = **JSX에서 태그를 그 자리에 적으면** 거기 들어간다. HTML과 똑같다.

```jsx
<div>
  <Header />    {/* 맨 위 */}
  <Menu />      {/* 그 아래 */}
  <Content />   {/* 또 그 아래 */}
  <Footer />    {/* 맨 밑 */}
</div>
```

순서 바꾸기 = 태그 줄을 위/아래로 이동. 가로 배치 = 부모에 `display: flex` (CSS 그대로). 결론: **HTML에서 `<div>` 적던 자리에 `<부품 />` 적으면 끝.**

### 1-5. useState — 드디어 state! (카운터 3연작)

React의 핵심, **state(상태) = 변하는 값, 바뀌면 화면 자동 갱신** 을 카운터로 입문했다.

```jsx
import { useState } from 'react'                 // 값 기억 기능 가져옴

const Counter = () => {
  const [count, setCount] = useState(0)          // count 보관(처음 0), 바꿀 땐 setCount
  return (
    <>
      <button onClick={() => setCount(c => c - 1)}>−1</button>
      <span>{count}</span>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
      <button onClick={() => setCount(0)}>리셋</button>
    </>
  )
}
```

**오늘의 핵심 문법:**

- `useState(0)` = 값 기억 시작 (괄호 안 = 초기값)
- `[count, setCount]` = **값 + 바꾸는 함수** 한 쌍
- ⚠️ 값 바꿀 땐 **꼭 set 함수로** — `count = count + 1`은 화면 갱신이 안 된다! (7일차 이벤트 카운터에서 `n++` 하고 textContent를 직접 고쳤던 그 일을, React에선 `setCount` 한 번이 대신한다)
- 버튼 여러 개(+1/−1/리셋)가 **한 state를 함께 조작** — 어느 버튼이든 `setCount` → 화면 자동 갱신
- `setCount(0)` = 초기값으로 리셋

## 2. 심화 학습 — 오늘은 쉬어가는 날

몸살 기운에 조퇴하느라 심화까지는 못 갔다. 하필 useState가 시작된 날 자리를 비워서 아쉬움이 크다 — state는 React의 심장이라던데, 카운터 뒷부분(폼 입력, props+state 조합)을 실습 없이 눈으로만 넘긴 게 마음에 걸린다. 대신 몸부터 회복하고, 놓친 부분은 수업 페이지로 따라잡기로 했다. 공부도 체력이 있어야 한다는 걸 배운 날이기도 하다. 💊
