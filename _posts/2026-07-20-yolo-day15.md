## 1. 수업 내용 요약 ① — Data (AI의 재료 준비)

AI가 배우려면 예시가 많이 필요하다 — 아이가 수많은 예시를 보며 "이게 강아지야"를 배우듯. 데이터를 구하는 4단계를 배웠다.

### 1-1. 데이터 수집 — 공개 데이터셋

인터넷에서 무료로 받는 방법. 대표 소스들:

| 소스 | 특징 |
|------|------|
| **AIhub** (aihub.or.kr) | 한국 정부 제공 — 한국어 음성·이미지·영상 |
| **Kaggle** / **Hugging Face** | 수백만 데이터셋 + ML 커뮤니티 |
| **COCO** | 33만 장·80개 카테고리 — YOLO 사전학습이 이걸로 됨 |
| **Roboflow Universe** | 컴퓨터 비전 특화, 1억+ 이미지 |
| **서울 열린데이터광장 / 공공데이터포털** | CCTV·교통 등 실시간 공공 데이터 |

### 1-2. 로컬 데이터 수집 — 웹캠으로 직접 촬영

공개 데이터에 없는 건 직접 찍는다. OpenCV(cv2) 첫 실전:

```python
import cv2, os
from datetime import datetime

os.makedirs("./captured_images", exist_ok=True)   # 폴더 없으면 자동 생성
cap = cv2.VideoCapture(0)                # 0 = 첫 번째 카메라 켜기
success, frame = cap.read()              # 사진 한 장 (success=성공 여부)
if success:
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")   # 파일명 안 겹치게
    cv2.imwrite(f"./captured_images/result_{timestamp}.jpg", frame)
cap.release()                            # 카메라 끄기 — 반드시!
```

- `cap.release()`를 빼먹으면 카메라가 켜진 채 남아 다른 프로그램이 못 쓴다 — 파일 입출력의 `with`처럼 "쓴 건 닫는다"
- 여러 장 찍기 = `cap.read()`를 반복문 안에 (어제 배운 for/while이 바로 실전에)

### 1-3. 데이터 증강 — 1장을 여러 장으로

학습 이미지가 적으면 AI가 특정 각도·밝기에만 익숙해진다. **회전·밝기·반전**으로 변형해 데이터를 불리는 게 증강(Augmentation):

```python
from PIL import Image, ImageEnhance, ImageOps

img = Image.open("image.png").convert("RGB")
img.rotate(90).save("./img_rotated.jpg")                      # 90도 회전
ImageEnhance.Brightness(img).enhance(0.5).save("./img_b.jpg") # 밝기 50%
ImageOps.mirror(img).save("./img_flip.jpg")                   # 좌우 반전
```

- 도구 구분: **파일 변환·저장 위주 = PIL / 실시간·카메라 = OpenCV**
- 어제 Teachable Machine이 잘 배웠던 비결 중 하나가 이런 증강이 내부에서 자동으로 돌기 때문이기도 하다 (YOLO도 기본 활성)

### 1-4. 데이터 라벨링 — "정답지" 만들기

이미지에 "여기가 사람"이라고 박스 좌표를 기록하는 작업. 도구는 **LabelImg**(로컬)·**Roboflow**(웹). YOLO 좌표 형식(.txt):

```
# class x_center y_center width height  (모두 0~1 정규화 비율)
0 0.525 0.376 0.284 0.418
```

- 한 줄 = 객체 하나, 클래스 번호는 0부터 (classes.txt의 줄 번호)
- **정규화**(픽셀 ÷ 이미지 크기) 덕분에 이미지 크기가 달라도 같은 형식

## 2. 수업 내용 요약 ② — YOLO 기초

### 2-1. YOLO란? — You Only Look Once

조셉 레드먼이 만든 오픈소스 객체 탐지 AI. **"한 번만 본다"** — 이미지를 번개처럼 훑어 모든 사물을 찾는다. 자율주행(차·사람·신호 인식), 로봇, 의료 진단 보조에 쓰인다. 10년 전엔 컴퓨터가 강아지·고양이도 헷갈렸는데 지금은 99%+ — 그 도약의 주역. (TED 강연도 시청: "How computers learn to recognize objects instantly")

### 2-2. YOLO 4대 기능 — 코드는 똑같고 모델명만 다르다

| 기능 | 모델 | 출력 |
|------|------|------|
| **분류** (Classification) | `yolo11n-cls.pt` | "이 사진은 강아지" — 사진 전체에 답 하나 |
| **탐지** (Detection) | `yolo11n.pt` | "여기 사람, 저기 차" — 위치 박스, 여러 개 동시 |
| **자세** (Pose) | `yolo11n-pose.pt` | 관절 17개 점 — 운동 자세·낙상 감지 |
| **분할** (Segmentation) | `yolo11n-seg.pt` | 박스가 아니라 픽셀 단위 외곽선 — 배경 제거 |

네 기능의 코드가 **완전히 동일**하다는 게 포인트:

```python
from ultralytics import YOLO
import cv2

model = YOLO("모델명")                 # ← 여기만 바꾸면 기능이 바뀜
results = model("이미지경로")
result_image = results[0].plot()       # 결과를 이미지에 그림
cv2.imwrite("저장경로.jpg", result_image)
```

- 모델 크기는 `n → s → m → l → x` — 작을수록 빠르고, 클수록 정확 (첫 트레이드오프)

### 2-3. 분류 모델 직접 학습 — 오늘의 하이라이트 ★

미리 학습된 `yolo11n-cls.pt`에 **내 클래스를 추가 학습**시켰다. 정확히는 어제 찾아봤던 전이학습의 한 기법인 **Fine-tuning** — pretrained 가중치를 출발점 삼아 새 데이터에 맞춰 미세 조정.

**데이터셋 3단 구성:**

```
dataset/
├─ train/   # 학습용 (70~80%) — 모델이 직접 보고 배움
├─ val/     # 검증용 (10~15%) — 매 epoch마다 "모의고사" 채점
└─ test/    # 시험용 (10~15%) — 최종 평가, 학습엔 절대 안 씀
```

- **train ↔ val 분리 필수** — 같은 이미지가 양쪽에 있으면 "시험 문제 유출", 점수가 부풀려짐
- val 성능이 가장 좋았던 시점이 **best.pt**로 자동 저장 (마지막 epoch인 last.pt 말고 이걸 쓰는 게 정석)

```python
model = YOLO("yolo11n-cls.pt")
model.train(data="dataset", epochs=2, batch=1, imgsz=256)
```

**과적합(Overfitting) — 오늘 배운 가장 중요한 개념:**

모델이 train 이미지를 **"외워버린"** 상태. 문제집을 통째로 암기한 학생 — 같은 문제는 100점, 살짝 바뀐 문제는 0점.

- **증상**: train 점수는 계속 오르는데 val 점수가 꺾여 내려감 (loss 곡선에서 val이 최저점 찍고 반등하는 지점 = 과적합 시작 신호, 그 직전이 best.pt)
- **원인**: epochs 과다 · 데이터 부족 · 데이터 대비 너무 큰 모델
- **대응**: epochs 줄이기/EarlyStopping(`patience`) · 데이터 증강 · **데이터 늘리기(가장 확실)** · best.pt 사용
- 반대는 과소적합(둘 다 낮음 — 덜 배운 상태)

**트레이드오프 — "가장 좋은 설정"은 없다:**

정확도↔속도(모델 크기), imgsz↑↔시간↑, batch↑↔메모리↑, epochs↑↔과적합 위험↑. 하드웨어·시간·목표에 맞는 **균형점을 실험으로 찾는 게 학습의 본질**이라는 말이 와닿았다.

**🏆 미션 — 표정 분류 챌린지:** Happy·Sad·Normal 3표정 분류 모델 만들기 (imgsz=256 고정, 클래스별 수량 통일, val accuracy로 시상!). Chrome의 Image Downloader 확장으로 이미지를 모으는 꿀팁도.

### 2-4. 추론 파라미터 튜닝 — 결과를 세밀하게 제어

```python
model = YOLO("yolo11n.pt")
print(model.names)        # 이 모델이 아는 클래스 목록 {0: 'person', ...}

model("input.jpg",
    save=True,            # 결과 자동 저장
    conf=0.5,             # 신뢰도 50% 이상만 (기본 0.25)
    max_det=3,            # 최대 탐지 수
    classes=[0, 2],       # 0=person, 2=car 만 탐지
    save_crop=True,       # 탐지 영역만 잘라 저장
    save_txt=True,        # 좌표 텍스트 저장
)
```

STEP 09 퀴즈("사람과 자동차만 탐지하려면?")의 답이 `classes=[0, 2]` — `model.names`로 클래스 번호를 확인하고 지정하는 방식. **이 부분을 더 견고하게 만든 게 오늘의 심화다 (아래에서!)**

### 2-5. HTTP 스트림 — 실시간 CCTV를 코드로 받기

웹캠 번호 대신 **m3u8 주소**를 넣으면 인터넷 CCTV가 열린다 (도시교통정보센터 UTIC의 공개 스트림 사용):

```python
cap = cv2.VideoCapture("https://.../playlist.m3u8")   # URL도 카메라처럼!
while cap.isOpened():
    success, frame = cap.read()
    if not success: break
    cv2.imshow("CCTV", frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):   # q 키로 종료
        break
cap.release()
cv2.destroyAllWindows()
```

- **m3u8 = HLS 재생목록** — 영상을 수 초 조각으로 쪼개 이어받는 방식 (유튜브 라이브·OTT도 동일)
- **`cv2.waitKey(1) & 0xFF == ord('q')` 해부** — waitKey(1)=1ms 동안 키 확인, `ord('q')`='q'의 번호(113), `& 0xFF`=키 번호를 0~255로 정리하는 안전장치. 컴퓨터에 따라 키 번호 앞에 쓸데없는 비트가 붙는데, **마지막 8비트만 잘라내면**(0xFF 마스킹) 어디서든 같은 값이 된다. 2진수 자릿값 계산(64+32+16+1=113)까지 손으로 따라가며 이해했다.
- **putText**로 프레임 위에 글자도 얹었다 — 주의: OpenCV 색상은 RGB가 아니라 **BGR 순서**! `(255,0,0)`이 파랑이다.

## 3. 어려웠던 것 — 심화 학습

### 딕셔너리 컴프리헨션 — model.names 뒤집기

STEP 09의 `classes=[0, 2]`처럼 번호를 하드코딩하는 대신, **클래스 이름으로 지정**하고 싶어서 Claude Code에게 코드 작성을 부탁했다 (yolo_params.py):

```python
# 클래스 이름 -> 인덱스 변환
target_names = ["dining table", "vase"]
name_to_id = {name: idx for idx, name in model.names.items()}
target_ids = [name_to_id[name] for name in target_names]

model("input_params.jpg", save=True, classes=target_ids)
```

가운데 줄이 도무지 안 읽혀서 설명을 요구했고, 답을 정리하면:

**`model.names`는 {인덱스: 이름} 형태의 딕셔너리다:**

```python
{0: 'person', 1: 'bicycle', ..., 60: 'dining table', ..., 75: 'vase', ...}
```

**문제의 줄은 그 키와 값을 뒤바꾼 새 딕셔너리를 만드는 "딕셔너리 컴프리헨션":**

```python
name_to_id = {name: idx for idx, name in model.names.items()}
```

셋으로 쪼개 읽으면:

1. `model.names.items()` → (idx, name) 쌍을 하나씩 꺼냄 — 예: `(60, 'dining table')`
2. `for idx, name in ...` → 각 쌍을 idx, name 변수로 풀어줌
3. `{name: idx for ...}` → 반복하며 **name을 키, idx를 값**으로 하는 새 딕셔너리 생성

결과:

```python
name_to_id = {'person': 0, ..., 'dining table': 60, 'vase': 75, ...}
```

**왜 뒤집나?** 원래 `model.names`는 인덱스→이름(정방향) 조회용이다. 나는 반대로 `name_to_id["dining table"]`처럼 **이름→인덱스** 조회가 필요해서 **역딕셔너리**를 만든 것.

**추가로 알게 된 것들:**

- 바로 다음 줄 `[name_to_id[name] for name in target_names]`는 **리스트 컴프리헨션** — 같은 문법의 리스트판. `{}`면 딕셔너리, `[]`면 리스트가 나온다.
- 컴프리헨션 = "반복문을 한 줄에 눌러 담아 **새 컬렉션을 만드는** 파이썬 특유의 문법". JS로 번역하면 `Object.entries(names).map(...)`을 한 줄로 쓰는 느낌인데, 파이썬 쪽이 훨씬 간결하다.
- 실용적 이점: 번호 하드코딩(`classes=[60, 75]`)은 모델이 바뀌면 깨질 수 있지만, **이름으로 지정하면 코드가 스스로 번호를 찾는다** — 읽기도 좋고 견고하다.

```python
# 일반 반복문으로 풀어 쓰면 (같은 동작)
name_to_id = {}
for idx, name in model.names.items():
    name_to_id[name] = idx
```

풀어 쓴 버전과 나란히 놓고 보니 구조가 보였다 — **컴프리헨션은 "결과물 모양을 먼저 쓰고, 반복을 뒤에 쓰는" 어순**이다.
