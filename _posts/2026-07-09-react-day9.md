웹개발 수업 9일차, 새 과목 **React** 시작. "화면을 레고 블록(컴포넌트)처럼 조립하는 도구"를 처음 만졌다. Vite로 프로젝트를 만들고, JSX 문법과 컴포넌트, props까지. 심화에서는 텍스트를 색상 코드로 바꾸는 `textToHex` 함수를 뜯어봤다.

## 1. 수업 내용 요약

### 1-1. React란? — 그리고 왜 쓰나

**React = 화면을 레고 블록(컴포넌트)처럼 조립해서 만드는 도구.** 숫자나 글자(데이터)만 바꾸면 화면은 React가 알아서 바꿔준다 — 내가 화면 고치는 코드를 안 써도 됨.

- **재사용** — 버튼·카드 블록을 한 번 만들면 어디서든 다시 사용
- **자동 갱신** — 값이 바뀌면 화면이 스스로 따라옴
- **표준** — 인스타그램·넷플릭스도 React로 제작

**"왜?"가 와닿았던 좋아요 버튼 비교:** 좋아요 숫자가 화면 3곳에 표시된다면 —

| | 기존 JS | React |
|---|---------|-------|
| 표시 자리 N개 추가 | HTML에 칸 + JS에 갱신 명령 = **N×2군데** 수정 | `{likes}` **N번** 쓰면 끝 |
| 한 곳 빠뜨리면 | 그 자리만 0에서 멈춤 (에러도 없음!) | 빠뜨릴 두 번째 절차가 없음 |

미니 프로젝트에서 `render()` 함수를 직접 만들어 화면을 갱신했던 게 기억났다 — **React는 그 render를 자동으로 해주는 도구**였다.

### 1-2. JSX — JS 안에 HTML 쓰는 규칙

JSX = `return ( )` 안에 HTML처럼 쓰는 문법. 거의 HTML과 같지만 **규칙 3개**만 다르다:

| HTML | JSX | 이유 |
|------|-----|------|
| `class` | `className` | JS 예약어 class와 겹침 |
| `for` | `htmlFor` | JS for문과 겹침 |
| `<br>` | `<br />` | 빈 태그도 꼭 닫기 |

`{ }` 안에는 **JS 값**을 넣는다 — 변수·계산·함수 호출 모두 가능. 템플릿 리터럴의 `${ }`와 같은 감각.

```jsx
<p>{name}님은 {age}살, 내년 {age + 1}살</p>
```

**JSX 파일 읽는 법 — 모든 .jsx는 같은 3층 구조:**

```jsx
import './App.css'          // ① 1층: 재료 가져오기
const App = () => {         // ② 2층: 컴포넌트 함수
  return ( ...화면... )     // ③ 3층: 화면 그리기 (JSX)
}
export default App          // ④ 이 부품 내보내기
```

### 1-3. 환경 준비 — Node.js + Vite

React를 만드는 도구들(npm·Vite)이 전부 Node.js 위에서 돌아간다. 비유: 완성된 장난감(React 앱)은 아이(브라우저)가 갖고 놀지만, **장난감 찍어내는 공장 기계(Node.js)**가 없으면 못 만든다.

프로젝트 생성은 명령어 하나 (선택지 주의!):

```bash
npm create vite@latest my-app
# Select a framework:  → React
# Select a variant:    → JavaScript (TypeScript 아님!)
# Use ESLint instead of Oxlint? → Yes (ESLint — 표준 린터·자료 많음)
# Install with npm and start now? → Yes (부품 설치 + 실행까지 자동)
```

**Vite(비트)가 하는 일 3가지:** ① 프로젝트 틀 자동 생성 ② 브라우저가 못 읽는 JSX를 저장할 때마다 순수 JS로 실시간 번역 ③ localhost:5173 미리보기 서버 (저장하면 즉시 반영 = HMR)

### 1-4. 프로젝트 구조 해부 — 빈 액자, 다리, 그림

my-app 폴더의 파일들을 하나씩 뜯어봤다. 핵심은 **연결 흐름**:

```
index.html (빈 액자) → main.jsx (다리) → App.jsx (그림)
```

| 파일 | 역할 | 손대나? |
|------|------|--------|
| `index.html` | **빈 액자** — `<div id="root">` 빈 칸 하나뿐 | 거의 안 고침 |
| `src/main.jsx` | **다리** — root 칸을 찾아 App을 끼워 넣음 | 거의 안 고침 |
| `src/App.jsx` ★ | **매일 그리는 그림** — 화면 짜고 부품 조립 | 매일 여기서 작업 |
| `src/App.css` | App 전용 꾸미기 / `index.css` = 전체 공통 | 필요할 때 |
| `node_modules/` | 설치된 부품 창고 | **절대 손대지 않음** (git 제외) |

main.jsx의 핵심 한 줄: `createRoot(document.getElementById('root')).render(<App />)` — JS에서 배운 `getElementById`가 여기 있었다!

### 1-5. 매일 작업하는 법

- 다음 날 다시 시작: 생성·설치 다시 안 함 — **`cd my-app` → `npm run dev`** 두 줄이면 끝
- 작업 흐름: **components/*.jsx**에 부품 만들기 → **App.jsx**에서 import해 조립 → 저장하면 자동 반영
- 서버 끄기: 터미널에서 `Ctrl + C`

**자주 막히는 것 (미리 기록):**

| 증상 | 원인 → 해결 |
|------|------------|
| 브라우저 빈 화면 | `export default App` 빠짐 (에러도 안 남!) |
| 수정했는데 안 바뀜 | 저장 안 함, 또는 서버 꺼짐 → `npm run dev` |
| 빨간 에러 화면 | 태그 안 닫음 등 — 에러에 적힌 파일명·줄번호로 이동 |

### 1-7. 컴포넌트 — 화면을 찍어내는 도장

컴포넌트 = 화면 조각을 찍어내는 **도장**. 함수 하나가 곧 부품 하나다.

```jsx
// ① src/components/Hello.jsx — 부품 만들기
const Hello = () => {
  return <h1>안녕!</h1>
}
export default Hello

// ② App.jsx — 조립하기
import Hello from './components/Hello.jsx'
const App = () => {
  return (
    <div>
      <Hello />   {/* 도장 쾅 */}
      <Hello />   {/* 또 쾅 — 복붙 아님 */}
    </div>
  )
}
```

**핵심 규칙 3가지:**
1. 이름은 **대문자**로 시작 (소문자면 일반 HTML 태그로 오해)
2. 하나의 컴포넌트는 **태그 하나로 감싸** 반환 — div 대신 흔적 없는 `<> </>`(Fragment, 투명 랩)도 가능
3. 맨 아래 `export default` 필수 — 빠지면 빈 화면 (에러도 안 남)

꾸미기 정석은 부품 옆에 같은 이름 CSS(`Hello.css`)를 두고 부품 맨 위에서 `import './Hello.css'`. React는 `class`가 아니라 `className`으로 연결한다.

### 1-8. Props — 부모가 자식에게 넘기는 값 (5종 세트)

props = 컴포넌트에 끼우는 입력값 (HTML 속성처럼 생김). **부모 → 자식 일방향, 읽기 전용.**

**① 기본 (Greeting):**

```jsx
const Greeting = ({ name }) => <h1>안녕, {name}!</h1>   // 구조분해로 바로 꺼냄

<Greeting name="React" />
<Greeting name="지니" />       // 같은 부품, name만 바꿔 재사용
```

**② 기본값 (Profile):** 안 넘겨도 안 깨지게 —

```jsx
const Profile = ({ name, job = "개발자" }) => ...
// job 생략하면 자동으로 "개발자"
```

**③ 여러 개 (Card):** `{ title, desc, emoji }` 쉼표로 나열. 넘길 땐 이름=값이라 **순서 상관없음** — 이름으로 매칭.

**④ 조건부 렌더링 (Avatar):**

```jsx
{online && <span>🟢</span>}   // online이 true면 그림, false면 아무것도 안 그림
```

**⑤ 삼항 연산자 (Badge):**

```jsx
const color = type === 'new' ? 'green' : 'crimson'   // "조건 맞아? 그럼 A, 아니면 B"
return <span style={{ background: color }}>{text}</span>
```

`style={{ }}`가 이중 중괄호인 이유: 바깥 `{}`는 "JS 값 넣기", 안쪽 `{}`는 **객체**. JSX 인라인 스타일은 객체로 넘긴다.

## 2. 어려웠던 것 — 심화 학습

### textToHex — 텍스트에 고유 색상 입히기

Profile 컴포넌트를 꾸미다가, 직업(job) 텍스트마다 **자동으로 고유한 색**을 입히고 싶어서 만든(배운) 함수. 처음 보는 연산자가 나와서 한 줄씩 해부했다.

```jsx
const textToHex = (text) => {
  let hash = 0;
  for (let i = 0; i < text.length; i++) {
    hash = text.charCodeAt(i) + ((hash << 5) - hash);
  }
  const hex = (hash & 0xFFFFFF).toString(16).padStart(6, '0');
  return `#${hex}`;
}
```

**동작 원리 4단계:**

1. **텍스트의 각 글자를 숫자로 변환** — `charCodeAt(i)`가 글자의 코드 번호를 돌려줌
2. **그 숫자들을 섞어서 하나의 큰 숫자(해시)를 만들기** — `(hash << 5) - hash`는 `hash × 32 − hash = hash × 31`과 같다. 비트를 왼쪽으로 5칸 미는 `<<`가 ×32인 것. 31 같은 소수를 곱하면 글자 순서만 달라도 결과가 잘 흩어진다(고전적인 문자열 해시 기법).
3. **16진수(HEX)로 바꾸기** — `hash & 0xFFFFFF`로 색상 범위(24비트)만 남기고, `.toString(16)`으로 16진수 문자열 변환, `.padStart(6, '0')`으로 6자리를 못 채우면 앞을 0으로 채움
4. **앞에 `#`을 붙여 색상 코드 완성**

**특징:**

- **일관성** — 같은 텍스트는 항상 같은 색상 생성 (랜덤이 아니라 해시라서!)
- **다양성** — 다른 텍스트는 다른 색상 생성
- **컬러 범위** — 항상 유효한 HEX 코드 (#000000 ~ #FFFFFF)

**단어별로 색 입히기 — JSX에 적용:**

```jsx
const textWithColors = job.split(' ').map((word, index, arr) => (
  <span key={index}>
    <span style={{ color: textToHex(word) }}>
      {word}
    </span>
    {index < arr.length - 1 && ' '}
  </span>
));
```

1. `split(' ')` — 문장을 띄어쓰기로 나누기
2. `map`으로 각 단어마다 다른 색상 생성 — `map`의 콜백이 (값, 인덱스, **원본 배열**) 세 개를 받는다는 것도 여기서 알았다
3. `{index < arr.length - 1 && ' '}` — 마지막 단어가 아니면 뒤에 띄어쓰기 추가. 오늘 배운 **`&&` 조건부 렌더링**이 "공백 한 칸"에도 쓰인다!

3일차 심화에서 "정확한 색은 hex 코드로"를 배웠는데, 이제 그 hex 코드를 **함수로 만들어내는** 단계까지 왔다.
